## Key technical findings

1. **`bed_idx` reset (ModelArrange.cpp:98)** is the architectural reason Phase 29 exists: arrange_objects resets per-bed indices, so the per-plate split MUST be reconstructed from translations via computePlateIndex.
2. **`computePlateIndex` sign-flip** — the upstream `(stride_y - t_y)/stride_y` form operates on the pre-shifted ArrangePolygon translation space; Qt6 reads raw world offsets so the correct decode is `-world_y/stride_y`. The unit test caught an initial bug here (the deterministic-test pattern working as intended).
3. **The 3MF writer (`store_bbs_3mf`) is the single shared blocker** for THUMB-02 + FIXTURE-02 — its serialization/pixel path throws on the Qt6 mock pipeline. Deferred to v3.3+ where real GL capture unblocks the full path.
4. **Auto-arrange to NEW plates** (vs preserving 3MF-loaded cross-plate state) needs the upstream virtual-bed mechanism, which the Qt6 `arrange_objects` overload doesn't expose. The rebuild's value is preserving loaded cross-plate state; auto-distribution to new plates is a v3.3+ enhancement.
