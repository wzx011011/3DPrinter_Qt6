## Non-blocking Gaps

### PLATE-09 test-fixture gap (the only genuine one)
The 3MF write/load code is correct and verified by build + code inspection (mirrors upstream `PartPlate.cpp:6160` store_to_3mf_structure + read_from_archive). The deterministic round-trip test is QSKIP'd because `store_bbs_3mf` requires valid model geometry the test harness cannot synthesize via `addPrimitiveToPlate` (its HAS branch creates an object not wired for 3MF IO in tests). A real-model fixture (committed test .3mf/.stl) would fully close PLATE-09. **Highest-priority test debt for v3.1.**

### Code-quality notes (not gaps, maintenance risks)
1. **clonePlate comment inaccuracy** (`ProjectServiceMock.cpp:1080`): says "inserts at sourceIndex+1" but `duplicateObject` appends at end. Code is correct (uses returned `dupIdx`), but the comment is a maintenance trap.
2. **config.apply merge direction untested** (`SliceService.cpp:393`): plate-wins-over-preset is asserted in comment only; Phase 19 tests verify the merge SOURCE, not the merged-result ordering. Low risk (DynamicPrintConfig::apply semantics are well-documented) but uncovered.
