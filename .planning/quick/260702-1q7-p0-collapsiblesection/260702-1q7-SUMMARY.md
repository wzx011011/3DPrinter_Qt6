# Quick Task 260702-1q7 — P0 UI fixes (sidebar + param table)

**Result:** All three P0 bugs fixed, build verified.

## Root causes & fixes

### 1. CollapsibleSection吞点击 — `14c4ca4`
`src/qml_gui/components/CollapsibleSection.qml` — the title-bar toggle `MouseArea`
was declared *after* the `RowLayout` (which hosts `rightActions`) and used
`anchors.fill: parent`, so it sat on top and swallowed clicks on every injected
right-action button (printer gear, filament auto-match, compat dot). Moved it
*before* the RowLayout (underneath). Action buttons' own MouseAreas now receive
their clicks; the rest of the title bar still toggles.

### 2. 耗材编辑按钮 — `2685a50`
`src/qml_gui/panels/LeftSidebar.qml` — the filament section had no edit button
(printer section has one; upstream has `m_edit_filament_btn`). Added a ✎ button
to `filamentSection.rightActions`, gated by
`presetActionBlocker(1, currentFilamentPreset, "rename")`, calling
`backend.openSettings()` — mirroring the printer edit button for the filament
tier. Clickable because of fix #1.

### 3. 在参数表中编辑隐藏 — `11103b0`
`src/qml_gui/components/ParamsPage.qml` — double/int option rows showed the value
as a read-only `Text`; only the slider could change it. Replaced with an editable
`TextInput` (Rectangle + `DoubleValidator`) following the Sidebar.qml
transform-input pattern: writes back via `optionModel.setValue(optIdx, v)` on
`editingFinished`, clamped to `[oMin, oMax]`, respecting `oRO`. Slider kept.
