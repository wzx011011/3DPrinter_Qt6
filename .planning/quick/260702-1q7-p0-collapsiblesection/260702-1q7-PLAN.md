# Quick Task 260702-1q7 — P0 UI fixes (sidebar + param table)

Three coherent P0 bugs in the Prepare sidebar / Settings parameter table. Bug #1 must be fixed for Bug #2's new button to be clickable.

## Task 1 — CollapsibleSection吞点击 (swallowed right-action clicks)

**File:** `src/qml_gui/components/CollapsibleSection.qml`
**Root cause:** Lines 84-88 — the title-bar `MouseArea` is declared *after* the `RowLayout` (lines 45-82) and uses `anchors.fill: parent` (titleBar). Later siblings sit on top in QML, so the MouseArea overlays the whole title bar including `actionsRow` (where `rightActions` buttons live) and eats their clicks. Visible victims: printer gear (LeftSidebar rightActions), filament auto-match + compatibility dot.
**Fix:** Move the toggle `MouseArea` to be declared *before* the `RowLayout` inside `titleBar` (underneath it). Action buttons keep their own MouseAreas on top → they receive clicks; non-button title-bar areas pass through the RowLayout/Text (mouse-transparent) to the toggle underneath → expand/collapse still works. No behavior change for consumers that don't use rightActions.
**Verify:** Click printer gear / filament auto-match buttons → they fire (openSettings / autoMatchFilament). Click title text → section still collapses/expands.

## Task 2 — 耗材编辑按钮 (filament edit-preset button)

**File:** `src/qml_gui/panels/LeftSidebar.qml`
**Root cause:** Filament section (lines 158-216) has no edit button. Printer section has one (lines 99-121, ✎ → backend.openSettings()). Upstream OrcaSlicer Sidebar provides `m_edit_filament_btn`; missing here, so users cannot edit the filament preset from the sidebar.
**Fix:** Add a ✎ button as the first entry of `filamentSection.rightActions`, mirroring the printer edit button but gated on the filament tier: `presetActionBlocker(1, configVm.currentFilamentPreset, "rename")`. `onClicked: backend.openSettings()`. (Tier targeting is not wired today — the printer button has the same behavior, so consistency holds; deferred nicety.)
**Verify:** ✎ button visible in filament section header; disabled/dim for builtin filament presets; opens Settings page when enabled. Relies on Task 1 fix to actually receive clicks.

## Task 3 — 在参数表中编辑隐藏 (editable numeric values in param table)

**File:** `src/qml_gui/components/ParamsPage.qml`
**Root cause:** Lines 324-330 — for double/int option rows, the value is rendered as a read-only `Text` (`Number(oVal).toFixed(...)`). Only the slider can change the value; direct keyboard entry is impossible ("edit hidden"). Upstream uses an editable SpinCtrl.
**Fix:** Replace the read-only `Text` with an editable `TextInput` inside a styled `Rectangle`, following the Sidebar.qml transform-input pattern (lines 177-212): `DoubleValidator` (decimals 3 for double / 0 for int), `selectByMouse`, `readOnly: paramRow.oRO`, write back on `onEditingFinished` via `root.optionModel.setValue(paramRow.optIdx, v)` clamped to `[oMin, oMax]`. Slider stays alongside.
**Verify:** Type a numeric value into a double/int param field, press Enter → value applied (slider moves, dirty marker appears); read-only params stay non-editable.
