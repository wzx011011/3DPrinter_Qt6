# v3.6 Screenshot and Source-Truth Starting Inventory

**Created:** 2026-07-01
**Milestone:** v3.6 Screenshot-Driven OrcaSlicer UI Restoration

## Purpose

This file seeds Phase 50. It records the initial screenshot inputs, upstream source candidates, Qt target candidates, and required mapping format for screenshot-driven restoration.

Screenshots define visual/layout truth. OrcaSlicer source defines behavior truth. A visible control is not complete until both sides are mapped.

## Required Mapping Format

Each Phase 50 module row should use this schema:

| Field | Meaning |
|---|---|
| Screenshot | Source screenshot filename. |
| Region | Visual region or control group. |
| Visible controls | Controls, labels, panels, sliders, buttons, menus, or indicators visible in the screenshot. |
| Upstream source | Exact OrcaSlicer files/classes/functions that define behavior. |
| Qt target | Existing or planned Qt/QML/C++ target. |
| Decision | Modify existing, replace existing, create new, or blocked. |
| Status | Real, Hybrid, Mock, Blocked, Placeholder, or Superseded. |
| Verification | Automated test, visual check, manual UAT, renderer harness, or source audit. |
| Cleanup | Old files/routes/resources/tests to delete when replacing. |
