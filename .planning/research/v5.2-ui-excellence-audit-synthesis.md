## Master gap list (deduplicated across all 6 audits)

### Bucket D1 — Design-system foundation (Theme.qml + missing tokens)

- **D1.1** Add missing color tokens: `statusErrorDark`, `statusErrorPressed`, `accentSubtlePressed`, `scrollBarColor`, `borderActive` (referenced but undefined → silent `undefined`). *(Controls audit)*
- **D1.2** Add missing sizing tokens (11): `sliderTrackHeight`, `switchTrackWidth`, `dialogHeaderHeight`, `dialogFooterHeight`, `controlHeightXL`, `panelPaddingSM`, `sidebarWidthMin`, `sidebarWidthMax`, `sidebarWidthDefault`, `rightPanelWidthMin`, `rightPanelWidthMax`. *(Controls + Panels audits)*
- **D1.3** Add `fontMono` token (eliminate 26 `font.family: "Consolas"` hardcodes across 8 component files). *(Components audit)*
- **D1.4** Add `fontSize13` (or rename `fontSizeMD`) — size 13 used 17× in pages but isn't in the scale. *(Pages audit)*
- **D1.5** Add `statusSeverity{0..9}` palette to Theme — collapse the 3 private 10-level severity tables in ErrorBanner/ErrorToast/NotificationCenter. *(Components audit)*
- **D1.6** Kill dead tokens: `Theme.sidebarWidth=240` (never read) and `Theme.panelPadding=12` (never read) — or wire them. *(Panels audit)*
- **D1.7** Add 8 missing loading/disabled/empty-state tokens (spinner color, skeleton color, dim overlay). *(Panels audit)*

### Bucket D2 — Color hardcode → token migration

- **D2.1** PreparePage: 46 hardcoded hex literals → Theme.* tokens. *(PreparePage audit)*
- **D2.2** **PreferencesPage.qml is a token-system island** — 0 Theme.* refs, 129 hardcoded hex colors. *(Pages audit — BLOCKER)*
- **D2.3** Dialogs: 228 hex literals across 14 dialogs; accent drift `#22c564` vs `Theme.accent #18c75e`; `#141920` footer hardcoded 9×; `#1e2535` divider hardcoded 6×. *(Dialogs audit)*
- **D2.4** LeftSidebar private palette (6 colors at `LeftSidebar.qml:17-22`) bypassing Theme. *(Panels audit — BLOCKER)*
- **D2.5** ObjectList hardcoded hex literals + StatusBar "ignores Theme entirely". *(Panels audit)*
- **D2.6** PresetDiffDialog status badges `#1f8a4c/#b03a3a/#c98a1a` ignore Theme.statusSuccess/Error/Warning. *(PreparePage audit)*
- **D2.7** GLToolbars hardcoded viewport-chrome translucents. *(Components audit)*
- **D2.8** CxButton/CxIconButton use `Qt.darker()` runtime manipulation (3 sites) — bypass token system entirely. *(Controls audit — BLOCKER)*

### Bucket D3 — Typography hardcode → token migration

- **D3.1** PreparePage: 114/179 `font.pixelSize` are hardcoded literals (64%); 92× size 10. *(PreparePage audit)*
- **D3.2** Dialogs: 228 raw `font.pixelSize` literals vs 15 token refs; 7 distinct raw sizes (9/10/11/12/14/16/18) including 2 off-scale. *(Dialogs audit)*
- **D3.3** Pages: 17 distinct `font.pixelSize` values vs 6-token scale; size 13 used 17×. *(Pages audit)*
- **D3.4** **Off-scale sizes**: fontSizeXS-2=8 in CxSpinBox (below XS=10 floor); size 9 used in ObjectList, GLToolbars, dialogs. *(Controls + Components + Panels audits)*
- **D3.5** 26 `font.family: "Consolas"` hardcodes → Theme.fontMono. *(Components audit)*

### Bucket D4 — Spacing hardcode → token migration

- **D4.1** **Zero `Theme.spacing*` token usage** in PreparePage (3965 lines). *(PreparePage audit — worst pillar 1/4)*
- **D4.2** 4 of 11 page files use 0 Theme.spacing* tokens (main.qml, BBLTopbar, PreviewPage, PreferencesPage) — the shell itself bypasses spacing tokens. *(Pages audit)*
- **D4.3** `Theme.panelPadding=12` is unused. Outer margins are 5/8/10/14 with no system across panels. *(Panels audit)*
- **D4.4** Dialogs: 13 distinct widths from 280 to 736 (no breakpoints); outer padding 4 incompatible defaults (12/16/20/24); only 1 dialog uses Theme.spacing*. *(Dialogs audit)*
- **D4.5** Arbitrary `bottomMargin: 52` and 7+ distinct raw spacing values in PreparePage. *(PreparePage audit)*

### Bucket D5 — Sidebar width system (BLOCKER)

- **D5.1** **392px sidebar lock is 7-layer**: 4 QML files + 3 C++ constexpr (`BackendContext.h:467-469` where min==max==default==392). The DockableSidebar drag handle is a **visible no-op** (`qBound(392, w, 392)` discards every drag) — violates Phase 74 UI-SPEC "visible controls must not be no-op". *(Panels audit — BLOCKER)*
- **D5.2** PreparePage sidebar-width duplication with contradictory 280px comment. *(Panels audit)*
- **D5.3** Phase 74 UI-SPEC mandates "compact" — does not actually mandate 392px. 392 comes from a screenshot measurement comment in `BackendContext.h:467`. No compact-layout fallback exists. *(Panels audit)*

### Bucket D6 — Copywriting & language consistency

- **D6.1** **Dialog language split**: 21 dialogs use Chinese source, 3 use English (CreatePresets, PresetDiff, FilamentGroupPopup). Project positions ZH-in-qsTr as source language. *(Dialogs audit — BLOCKER)*
- **D6.2** Pages language inconsistency: MultiMachinePage entirely English (55 EN qsTr), CalibrationPage splits EN/zh within same row, Monitor/Home/main are zh. Tabs visibly flip language. *(Pages audit)*
- **D6.3** PreparePage info bar mixes Sliced/Stale/Ready/OK/non-manifold edges into Chinese chrome. *(PreparePage audit)*
- **D6.4** Phase 158 Emboss tooltips leak dev jargon (`Emboss.hpp`/`ProjectCurve`/上游/持久化) into user copy at PreparePage.qml:3342-3351. *(PreparePage audit)*
- **D6.5** Untranslated "LAN"/"CP……" strings. *(Pages audit)*
- **D6.6** Zero `qsTr()` wrappers in entire Cx* controls library (violates AGENTS.md). *(Controls audit)*

### Bucket D7 — Visual consistency (pseudo-buttons + raw controls)

- **D7.1** **8 dialogs render empty header bar** — set `title:` instead of `dialogTitle:`. CxDialog.qml:14 explicitly suppresses `title: ""`, so these render a 44px header with only ✕. Files: CreatePresets, ExportPresetBundle, NetworkTest, PresetDiff, SavePreset, SelectMachine, Troubleshoot, UnsavedChanges. *(Dialogs audit — BLOCKER)*
- **D7.2** Massive `Rectangle+Text+MouseArea` pseudo-button pattern bypassing Cx* library: MonitorPage 92 instances, MultiMachinePage 51, CalibrationPage 27, PreparePage gizmo action rows. *(Pages + PreparePage audits)*
- **D7.3** 8 hand-rolled `Rectangle+Text+MouseArea` pseudo-buttons in dialogs → CxButton. *(Dialogs audit)*
- **D7.4** Button-row inconsistency: 5 dialogs put affirmative first, 6+ put cancel first; alignment varies. *(Dialogs audit)*
- **D7.5** Phase 158 boldness uses raw `Slider` while every peer uses `CxSlider`. *(PreparePage audit)*
- **D7.6** Modal policy chaos: 8 modal / 11 NoAutoClose / 1 conditional / 1 escape-only / 1 NonModal / 2 unset. *(Dialogs audit)*
- **D7.7** Card-in-card nesting violates Phase 74. *(Panels audit)*

### Bucket D8 — Component coherence

- **D8.1** **3 duplicated notification systems**: ErrorBanner/ErrorToast/NotificationCenter — 3 private copies of the same 10-level severity→color table (~50 duplicated hex literals); sev=1 renders in both banner and toast simultaneously. *(Components audit — BLOCKER)*
- **D8.2** **4 orphan components registered in qml.qrc with zero consumers**: CxPanel, CxSectionHeader, FilamentSlot, GroupNavSidebar. *(Components audit — BLOCKER)*
- **D8.3** LeftSidebar vs DockableSidebar: two sidebar implementations, possibly redundant. *(Panels audit)*
- **D8.4** MoveSlider/PreviewLayerRail: parallel step-button idioms not unified. *(Components audit)*

### Bucket D9 — Control-library quality

- **D9.1** CxButton has no press-scale (other controls do); CxButton/CxPillAction have no focus border; only CxIconButton exposes ToolTip. *(Controls audit)*
- **D9.2** Disabled-state pattern inconsistent — two patterns (opacity 0.45 + color branches), some controls apply both. *(Controls audit)*
- **D9.3** 4 FAIL-class controls: CxPillAction, CxScrollView, CxSpinBox, CxDialog (near-miss). *(Controls audit)*
- **D9.4** `fontSizeXS - 2` in CxSpinBox produces 8px text, below XS=10 floor. *(Controls audit)*

### Bucket D10 — Experience safety

- **D10.1** **Destructive actions fire with zero confirm**: PreparePage deleteSelection/removeAllOnPlate/deletePlate/clearAllPaintData; CaliHistoryDialog "清空"; HomePage cloudUnbindDevice; MultiMachinePage removeDevice/stopAllLocalTasks/stopAllCloudTasks; MonitorPage disconnectDevice; ObjectList bulk delete. *(PreparePage + Dialogs + Pages + Panels audits — BLOCKER-class)*
- **D10.2** Async Emboss has no progress/spinner affordance. *(PreparePage audit)*
- **D10.3** SliceProgress missing indeterminate/canceled/error states. *(Panels audit)*
- **D10.4** No loading states anywhere in components layer. *(Components audit)*
- **D10.5** Uneven empty-state coverage across components. *(Components audit)*
- **D10.6** Weak login error UX; silent save-failure paths in ProjectPage. *(Pages audit)*
- **D10.7** Positive: strong disabled-state coverage (51 `enabled:` bindings in PreparePage); MonitorPage has complete 4-state machine with animated spinner; PreviewPage/CalibrationPage/HomePage/MultiMachinePage have proper empty/loading states.

## Phase mapping (for v5.2 ROADMAP)

The natural phase boundaries (each phase is a coherent, parallelizable workstream):

1. **Phase 160 — Theme token foundation** (Bucket D1). Add the ~25 missing tokens, kill dead ones, document the canonical token list. Foundation for all later phases.
2. **Phase 161 — Cx* control-library hardening** (Bucket D9 + control-specific D2.8/D3.4). Fix Qt.darker(), unify disabled pattern, backfill states, fix CxSpinBox 8px. Foundation for app-wide sweeps.
3. **Phase 162 — Color hardcode sweep** (Bucket D2). Mechanical replacement across pages/dialogs/components/panels. Parallel-safe.
4. **Phase 163 — Typography hardcode sweep** (Bucket D3). Same mechanical-replacement pattern, parallel-safe with 162.
5. **Phase 164 — Spacing hardcode sweep + sidebar width fix** (Bucket D4 + D5). The 7-layer sidebar lock + introduce Theme.spacing* usage everywhere.
6. **Phase 165 — Copywriting & language sweep** (Bucket D6). Pick ZH as source, sweep EN dialogs, kill dev jargon, add qsTr() to Cx* library.
7. **Phase 166 — Dialog consistency repair** (Bucket D7). Fix the 8 empty headers, replace pseudo-buttons, standardize button-row + modal policy.
8. **Phase 167 — Component coherence** (Bucket D8). Collapse notification trio → NotificationCard base; delete/wire 4 orphans; unify step-button idiom.
9. **Phase 168 — Visual control migration** (Bucket D7.2/D7.5). Replace Rectangle+MouseArea pseudo-buttons across pages with CxButton/CxIconButton; Emboss Slider → CxSlider.
10. **Phase 169 — Experience safety: destructive-action confirms + async affordances** (Bucket D10). Confirmation dialog component + route 12+ destructive triggers; spinner on async Emboss; SliceProgress state coverage.
11. **Phase 170 — REGRESS-06 cross-workstream UI regression gate**. Lock every contract from Phases 160-169 via source-audit slots.

11 phases (160-170). Foundation phases 160-161 are serial; 162-169 are parallelizable in waves; 170 is the tail.
