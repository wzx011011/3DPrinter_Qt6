# Phase 73 Plan 01

## Objective

Retire the old OpenGL viewport implementation while preserving the existing
QML `GLViewport` compatibility alias for the RHI/default path.

## TDD / RED Step

1. Update `QmlUiAuditTests` so the intended retirement is locked before code
   removal:
   - `main_qml.cpp` must not mention `OWZX_OPENGL`.
   - `main_qml.cpp` must not include or register `GLViewport`.
   - `CMakeLists.txt` must not list `GLViewport`, `GLViewportRenderer`, or
     `GCodeRenderer` sources.
   - the deleted renderer files must be absent from disk.
   - `RhiViewport` remains registered as the QML type name `GLViewport`.
   - `SoftwareViewport` remains the non-RHI fallback registration.
2. Run the focused audit test and confirm it fails while old files/branch still
   exist.

## Implementation Steps

1. Update `src/qml_gui/main_qml.cpp`:
   - remove `#include "qml_gui/Renderer/GLViewport.h"`.
   - remove `useOpenGLViewport` and all `OWZX_OPENGL` branches.
   - keep `OWZX_RHI_RENDERER=auto` default.
   - select QRhi through `RhiBackendSelector`; if unavailable, set
     `QT_QUICK_BACKEND=software`.
   - register `RhiViewport` as `OWzxGL.GLViewport` when QRhi is usable,
     otherwise register `SoftwareViewport`.
   - update obsolete comments that reference `QQuickFramebufferObject`
     GLViewport destructor behavior.
2. Update `CMakeLists.txt`:
   - remove `GLViewport.cpp/.h`.
   - remove `GLViewportRenderer.cpp/.h`.
   - remove `GCodeRenderer.cpp/.h`.
3. Delete obsolete files:
   - `src/qml_gui/Renderer/GLViewport.cpp`
   - `src/qml_gui/Renderer/GLViewport.h`
   - `src/qml_gui/Renderer/GLViewportRenderer.cpp`
   - `src/qml_gui/Renderer/GLViewportRenderer.h`
   - `src/qml_gui/Renderer/GCodeRenderer.cpp`
   - `src/qml_gui/Renderer/GCodeRenderer.h`
4. Update `QmlUiAuditTests`:
   - invert legacy assertions that required `OWZX_OPENGL` or `GLViewport.h`.
   - preserve assertions for RHI/Software registrations and QML compatibility.
   - add a dedicated deletion guard for the obsolete renderer source files.
5. Update source comments only where they incorrectly describe active runtime
   behavior. Historical extraction comments may remain if they are source
   provenance rather than active dependency claims.
