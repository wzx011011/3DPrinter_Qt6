<objective>
Lock in ASM-01's persistence guarantee with an automated round-trip test: write a known
per-instance assemble transform via the Plan 01 accessors, save the project through the
REAL `saveProjectAs` -> `Slic3r::store_3mf` path (ProjectServiceMock.cpp:1059-1086), reload
it through the real `loadProject` path, and assert the assemble transform survives intact.
This is the load-bearing proof for ASM-01 success criterion 2 ("round-trip through the
model: 3MF save -> reload preserves per-volume transforms") and must go through the real
libslic3r 3MF writer/reader, NOT a mocked store.

Depends on Plans 01-03: uses the Plan 01 accessors as both the write and the assert
interface; assumes Plan 02/03 wiring is in place so the feature is otherwise complete.
This plan is verification-only (test + CMake registration); it does not change product
code.
</objective>

<truths>
- T-01: The upstream `<assemble>` block round-trip already exists and is exercised by `store_3mf` (bbs_3mf.cpp:8070-8088, writes only when `is_assemble_initialized()`) and the loader (bbs_3mf.cpp:4734-4741, `set_assemble_from_transform` + `set_offset_to_assembly`). No new 3MF code is needed — the test merely proves the Qt layer's accessors (Plan 01) feed this contract correctly.
- T-02: `ProjectServiceMock::saveProjectAs` (cpp:1059-1086) calls `Slic3r::store_3mf(filePath, model_, &config, false, nullptr, true)` under `HAS_LIBSLIC3R`. `loadProject` reconstructs the `Slic3r::Model*` from the same file. This is the real round-trip path the test must use — do not mock `store_3mf`.
- T-03: The assemble transform is only serialized when `ModelInstance::is_assemble_initialized()` is true (bbs_3mf.cpp:8070). Calling `setAssembleOffset/Rotation/Scale` (Plan 01) flips this via the upstream setters (Model.hpp:1282, 1290), so a transform written through the Plan 01 setters WILL be serialized. The test must assert this implicitly by checking the reloaded value matches, AND can explicitly assert `isAssembleInitialized(idx)` is true after reload.
- T-04: The test must guard on `HAS_LIBSLIC3R`: without libslic3r there is no real 3MF path, so the round-trip cannot run. Use `#ifdef HAS_LIBSLIC3R` around the test body and `QSKIP` outside it, matching how other ViewModelSmokeTests cases handle libslic3r-dependent behavior.
- T-05: ViewModelSmokeTests links `owzx_app_core` (CMakeLists.txt:414-416) and is built with `HAS_LIBSLIC3R` in the canonical build, so the real 3MF path is available there. This is the correct test target — do not create a new test binary unless the round-trip needs facilities ViewModelSmokeTests cannot provide.
- T-06: The GL<->slic3r axis swap (Y/Z) and deg<->rad conversion live inside the Plan 01 accessors. The test should write and read through those accessors using GL-space values, and assert the GL-space round-trip is identity — this validates the accessors' convention handling end-to-end, not just the raw slic3r field.
- T-07: The AUTOMOC weak-dependency caveat applies to ViewModelSmokeTests (CMakeLists.txt:405-413, test file header): adding a new slot requires a cmake reconfigure to re-run AUTOMOC. The canonical verify script reconfigures; note this in the task action so an incremental builder knows to delete `build/ViewModelSmokeTests_autogen/timestamp` if skipping the full verify.
- T-08: Round-trip tolerance must account for 3MF's `float`/string serialization of the 4x3 matrix (bbs_3mf.cpp:8073-8080 writes matrix elements). Use a loose-but-meaningful epsilon (e.g. 1e-4) for the offset assertion, tighter for rotation degrees where the matrix->euler extraction can introduce small numeric drift.
</truths>
