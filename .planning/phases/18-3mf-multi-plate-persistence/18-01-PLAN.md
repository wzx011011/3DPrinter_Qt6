# Plan 18-01: 3MF Multi-Plate Persistence (store_to_3mf_structure write path + full load restore)

<objective>
Fix the v2.9-identified blocker: multi-plate state is lost on save. Build a `PlateDataPtrs` from `m_plateList` and feed it into `StoreParams.plate_data_list` before `store_bbs_3mf` (save path, PLATE-07); extend `loadProject`'s 3MF reconstruction to restore all plate state (locked, config, bed-type/sequence/spiral from config) — not just names+membership (PLATE-08); add a deterministic round-trip test proving the blocker is fixed (PLATE-09).
</objective>

<must_haves>
  <truth>saveProject builds PlateDataPtrs from m_plateList and sets StoreParams.plate_data_list before store_bbs_3mf</truth>
  <truth>Each PlateData carries plate_index, plate_name, locked, objects_and_instances (from PartPlate instance set), config (DynamicPrintConfig with bed-type/sequence/spiral keys)</truth>
  <truth>release_PlateData_list is called after store to free heap PlateData</truth>
  <truth>loadProject restores locked, config, and bed-type/sequence/spiral from PlateData — not just names+membership</truth>
  <truth>A deterministic round-trip test passes: save multi-plate project → reload → assert count/names/locked/bed-type/objects preserved</truth>
  <truth>canonical verify exits 0; existing tests don't regress</truth>
</must_haves>
