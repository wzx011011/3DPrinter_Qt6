# Plan 21-01: Review-Driven Bug Fixes (clonePlate + scoped-config)

<objective>
Fix 2 functional bugs found in the v3.0 code review, plus add tests that would have caught them. This is a scope-fence phase: no new features, only correctness fixes for already-shipped v3.0 code.

**BUG-1 (P0):** clonePlate reuses duplicateObject, whose mock-mode branch adds the new object to `currentPlate()` — but clonePlate never sets current=dst. Result: in mock mode, cloned objects land on the wrong plate (current, not dst) OR are double-assigned (current + dst). HAS mode unaffected (HAS branch doesn't set membership). Test missed it (only asserted dst has objects, not that current didn't gain any).

**BUG-2 (P0):** setPlateScopedOptionValue calls `cfg.option(k, true)` which returns nullptr for keys not in the config schema; the code returns false silently with no diagnostic. Per-plate overrides for typo'd/unregistered keys are silently dropped.

</objective>

<must_haves>
  <truth>clonePlate in mock mode adds cloned objects to dst (not current); currentPlate object count unchanged after clone</truth>
  <truth>setPlateScopedOptionValue on an unregistered key logs a warning (qWarning) rather than silent false</truth>
  <truth>clone test asserts currentPlate did not gain objects</truth>
  <truth>config-merge direction test added (plate override wins over preset after config.apply)</truth>
  <truth>canonical verify exits 0; existing 43 tests don't regress</truth>
</must_haves>
