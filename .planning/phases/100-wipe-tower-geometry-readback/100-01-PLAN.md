# Phase 100 Plan: Wipe-Tower Geometry Readback

## Objective

Implement the post-slice wipe-tower geometry readback so the Qt6 Prepare view
receives **real** wipe-tower dimensions from `Print::wipe_tower_data()` instead
of the hardcoded placeholder defaults (10/10/50/100/25).

Concretely, this plan:

1. Reads `Print::wipe_tower_data()` / `Print::has_wipe_tower()` inside the
   **SliceService worker**, right after `print.process()` succeeds and before
   the `Print` is invalidated. The geometry is captured **by value** into a
   small POD struct (`WipeTowerGeometry`) — no `Print*` escapes the worker.
2. Delivers the captured struct to the GUI thread via a new
   `wipeTowerGeometryReady` signal, alongside the existing `sliceFinished`
   path (`SliceService.cpp:763`).
3. Extends `EditorViewModel` to receive the struct and expose it as Q_PROPERTYs
   (`wipeTowerWidth/Depth/Height/X/Z` + `showWipeTower`) with a
   `wipeTowerGeometryChanged` notify signal — mirroring the existing
   `bedWidth`/`bedShapeChanged` pattern (`EditorViewModel.h:634-640,802`).
4. Binds `PreparePage.qml:1648` `GLViewport` wipe-tower Q_PROPERTYs to
   `editorVm.wipeTower*` so the real dims flow end-to-end through the renderer's
   `m_wipeTowerDirty` rebuild path (closes the `WT-VIEWPORT-DEFAULTS` unbound
   gap flagged in `99-GAP-MATRIX.md`).
5. Enforces the `has_wipe_tower()` gate (WTREAD-02): when false, no geometry is
   pushed and `showWipeTower` stays false (no placeholder box leak on
   single-material slices). Aligns `SoftwareViewport`'s divergent default
   (`m_showWipeTower=true` at `SoftwareViewport.h:231`) to the same
   data-driven gate so neither render path leaks a box.

Out of scope for Phase 100 (owned by Phase 101 / future, per `99-GAP-MATRIX.md`):
- The actual Option A rendering upgrade (feeding real dims into
  `buildWipeTowerVertices` is already the existing behavior; Phase 101 locks
  the renderer upgrade + brim/rib representation). Phase 100 only delivers real
  dims to the renderer-facing Q_PROPERTY layer — the renderer still draws
  whatever dims it receives.
- Option B real-mesh wipe-tower via `wipe_tower_mesh_data` / `convex_hull_3d`
  (future upgrade).

## Requirements

- `WTREAD-01`: After a successful slice, the Qt6 app reads real wipe-tower
  geometry from `Print::wipe_tower_data()` — at minimum the bbx (position,
  width, depth), height, brim_width, and rib_offset — replacing the hardcoded
  placeholder defaults.
- `WTREAD-02`: The readback respects `Print::has_wipe_tower()` — when no
  wipe-tower exists (single-material / `enable_prime_tower` off), no
  wipe-tower geometry is pushed and the renderer shows none (graceful, no
  placeholder box leak).
