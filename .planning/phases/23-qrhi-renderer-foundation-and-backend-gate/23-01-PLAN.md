# Plan 23-01: QRhi Backend Gate And Startup Fallback

<objective>
Introduce the app-side QRhi backend gate and fallback policy without changing the default stable viewport. This plan implements RHI-01, RHI-02, and PERF-05 at the startup/diagnostic layer: `OWZX_RHI_RENDERER=1` becomes a separate runtime gate, backend preflight attempts D3D12 then D3D11, diagnostics are structured, and failure falls back to `SoftwareViewport`.
</objective>

Implement D3D12-first then D3D11 fallback for Windows. Accept only known requested values (`auto`, `d3d12`, `d3d11`); invalid values must record a failure reason and fall back safely. Reuse the benchmark's QRhi preflight logic at a conceptual level, but keep app code independent from `tools/render_bench`. Do not include Vulkan in the default app candidate list with this SDK. Add the new files to `owzx_app_core` in `CMakeLists.txt`.

    <done>
`RhiBackendSelector` exists, compiles as part of `owzx_app_core`, records D3D12 and D3D11 attempts, rejects unknown env values, and exposes a selected `QSGRendererInterface::GraphicsApi` only when QRhi can be used.

    <action>
Update `main_qml.cpp` so renderer selection is resolved before `QGuiApplication app(argc, argv)`. Preserve the existing priority: `OWZX_OPENGL` remains the legacy OpenGL override; otherwise `OWZX_RHI_RENDERER=1` may select QRhi; otherwise set `QT_QUICK_BACKEND=software` and register `SoftwareViewport`. If QRhi preflight succeeds, call `QQuickWindow::setGraphicsApi(selectedApi)` and avoid setting `QT_QUICK_BACKEND=software`. If preflight fails, append diagnostics through `appendStartupLog()` and register `SoftwareViewport`. Do not show a modal or return a failure code from startup solely due to QRhi preflight failure.
    </action>
    <verify>
      <automated>rg -n "OWZX_RHI_RENDERER|setGraphicsApi|Direct3D12|Direct3D11|SoftwareViewport|appendStartupLog" src/qml_gui/main_qml.cpp</automated>
    </verify>
    <done>
Default no-env startup still sets software backend and registers `SoftwareViewport`; `OWZX_OPENGL` still registers legacy `GLViewport`; `OWZX_RHI_RENDERER` logs backend diagnostics and falls back to software if QRhi is unavailable.
    </done>
  </task>
