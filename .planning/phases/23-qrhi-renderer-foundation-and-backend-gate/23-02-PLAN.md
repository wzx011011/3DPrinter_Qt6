# Plan 23-02: QQuickRhiItem Viewport Host And Shader Pipeline

Create `RhiViewport : public QQuickRhiItem` and `RhiViewportRenderer : public QQuickRhiItemRenderer`. Mirror the existing `GLViewport`/`SoftwareViewport` property and enum surface needed by current QML: `canvasType`, `meshData`, `previewData`, layer/move controls, travel/bed/wipe tower toggles, marker properties, `gizmoMode`, `wireframeMode`, cut plane properties, and `lastThumbnailData`. For Phase 23, setters should store state and call `update()`; they do not need full real mesh/G-code rendering. The renderer should clear/render a small diagnostic primitive or color state using QRhi to prove the QML item pipeline is alive. Keep backend selection out of this class.

      <automated>rg -n "class RhiViewport|QQuickRhiItem|QQuickRhiItemRenderer|Q_PROPERTY\\(QByteArray meshData|CanvasPreview|GizmoMove" src/qml_gui/Renderer/RhiViewport* src/qml_gui/Renderer/RhiViewportRenderer*</automated>
    </verify>
    <done>
`RhiViewport` compiles as a QML item, exposes the existing viewport binding surface, and `RhiViewportRenderer` implements `initialize`, `synchronize`, and `render` using QRhi hooks.
    </done>
  </task>
