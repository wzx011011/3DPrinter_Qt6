<objective>
Complete the local FFF preset compatibility and selection-state layer so printer changes update compatible filament/process choices, preserve or repair valid selections, visibly block invalid combinations, and expose read-only action blockers through the C++ viewmodel/QML surface.

This plan intentionally replaces simplified legacy behavior when it conflicts with upstream compatibility semantics. Compatibility logic belongs in C++ services/viewmodels; QML only binds to the resulting state.
</objective>

<truths>
- D-01: Upstream OrcaSlicer `PresetBundle::update_compatible`, `Preset::is_compatible`, `PresetComboBoxes`, and `ConfigWizard` are the source-truth target for compatibility and selection repair.
- D-02: Printer changes preserve current filament/process only when compatible; otherwise repair to the first compatible preset in stable service order.
- D-03: When no compatible fallback exists, keep the selected preset visibly invalid and block slice/preview/export readiness rather than silently selecting an arbitrary preset.
- D-04: Unsupported upstream compatibility expressions are classified explicitly as warning/blocking state instead of silently accepted.
- D-05: Built-in/system/read-only presets must expose destructive-action blockers.
- D-06: QML must not contain compatibility business logic.
</truths>
