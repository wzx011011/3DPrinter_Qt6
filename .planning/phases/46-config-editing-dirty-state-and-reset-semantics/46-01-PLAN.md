<objective>
Replace the current partial config-editing implementation with an upstream-aligned preset editing model: typed values stay schema-driven, dirty/reset semantics become tier-aware instead of default-based, scope overrides remain safe, and preset/page/project transitions cannot silently discard unsaved edits.

This plan intentionally removes or bypasses wrong local semantics where they conflict with source truth. Phase 46 is the semantic foundation for Phase 47-49 preset lifecycle and slice integration work.
</objective>

<truths>
- D-01: Upstream dirty state is based on edited preset config vs selected/saved preset references, not vs schema defaults.
- D-02: Reset-one/reset-all for preset editing restore the selected preset value of the active tier unless the user explicitly requests another inheritance level.
- D-03: Scope override reset removes only the scope override and reveals the inherited effective value beneath it.
- D-04: Save/Discard/Cancel decisions for preset/scope/page/project transitions must be owned by C++ state, with QML dialogs acting only as presentation.
- D-05: Direct QML calls that bypass unsaved-change guards or reset semantics are invalid and should be replaced.
- D-06: Existing schema-driven option metadata and typed models should be reused; only the editing semantics need replacement.
</truths>
