# Phase 79 Plan: Preview Source-Truth Gap Audit

## Objective

Create the canonical v4.0 Preview page gap matrix. The matrix must let later
phases implement restoration without re-discovering which region, file,
upstream source, requirement, and verification path each gap belongs to.

## Tasks

1. Load target/current evidence:
   - `shotScreen/预览页.png`
   - current Preview QML/C++ implementation
   - v3.6 Phase 54 and Phase 55 Preview artifacts
   - Preview debug records for unusable controls and orbit disappearance

2. Map Preview regions:
   - `PV-TOP`
   - `PV-LEFT`
   - `PV-VIEWPORT`
   - `PV-RIGHT-LEGEND`
   - `PV-RIGHT-GCODE`
   - `PV-LAYER-RAIL`
   - `PV-MOVE-BAR`
   - `PV-COLOR-MODES`
   - `PV-STATS`
   - `PV-INTERACTION`
   - `PV-CLEANUP`

3. Read current Qt targets:
   - `src/qml_gui/pages/PreviewPage.qml`
   - `src/qml_gui/components/StatsPanel.qml`
   - `src/qml_gui/components/Legend.qml`
   - `src/qml_gui/components/VisibilityFilter.qml`
   - `src/qml_gui/components/LayerSlider.qml`
   - `src/qml_gui/components/MoveSlider.qml`
   - `src/core/viewmodels/PreviewViewModel.*`
   - `src/qml_gui/Renderer/RhiViewport*`

4. Read upstream anchors:
   - `GUI_Preview.*`
   - `GCodeViewer.*`
   - `GLCanvas3D.*`
   - `IMSlider.*`
   - `libvgcode/include/Types.hpp`
   - `libvgcode/include/Viewer.hpp`
   - `libvgcode/src/Settings.hpp`
   - `libslic3r/GCode/GCodeProcessor.*`

5. Produce `79-GAP-MATRIX.md` with:
   - target observation,
   - current evidence,
   - Qt target,
   - upstream source,
   - modify-vs-replace decision,
   - severity,
   - owner phase,
   - requirement ID,
   - verification method.

6. Verify the audit artifact:
   - all 11 Preview region IDs present,
   - all 13 v4.0 requirements referenced or explicitly out of scope,
   - every gap has owner phase, replacement decision, and verification method,
   - `git diff --check`,
   - encoding guard.

## Acceptance Criteria

- `79-GAP-MATRIX.md` exists and contains all 11 Preview region IDs.
- Every row maps to an upstream source and a Qt target.
- Every row has severity, owner phase, requirement ID, replacement decision,
  and verification method.
- v3.6/v3.7 residual Preview gaps are reconciled into v4.0 ownership.
- Phase 79 verification status is `passed`.
