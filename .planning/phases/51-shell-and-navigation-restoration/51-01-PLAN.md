# Plan 51-01: C++ Shell Action Gate Properties on BackendContext

<objective>
Add the shell-level action enablement/loading/blocked state that SHELL-03 requires to live in C++, not QML-only conditions. Add 8 aggregate Q_PROPERTY gate properties on BackendContext (`canImport`, `canSlice`, `isSlicing`, `canExport`, `canSave`, `canUndo`, `canRedo`, `isBusy`) plus 4 state-dependent blocked-reason label strings (`exportActionLabel`, `exportActionHint`, `saveActionLabel`, `saveActionHint`). Each gate forwards to the underlying EditorViewModel/PreviewViewModel source-of-truth property and re-emits via BackendContext signals. Also harden the SHELL-02 state-preservation contract: wire BackendContext to forward EditorViewModel/PreviewViewModel `stateChanged` through its own bulk `stateChanged()` so the shell never observes stale gate state across a Prepare↔Preview round-trip. This is the C++ foundation that Wave 2 QML (Plan 51-02) binds to, so it must land first.

This plan does NOT touch QML files (Wave 2), tests (Wave 3), or `components/Toolbar.qml` removal (Wave 3). It adds pure C++ state surface on the composition root only.
</objective>

C++ ONLY on the composition root. The only files modified are `BackendContext.h` and `BackendContext.cpp`.

// Phase 51 SHELL-02/SHELL-03: forward viewmodel stateChanged so shell gates
