# Plan 53-01: Prepare workflow C++ state and action gates

<objective>
Make `EditorViewModel` expose honest, source-truth-aligned state for Prepare
object, plate, viewport, and gizmo commands so QML does not guess business
rules.
</objective>

<truths>
- OrcaSlicer object operations are defined by `GUI_ObjectList.cpp` and
  `Plater.cpp`.
- OrcaSlicer plate operations are defined by `PartPlateList` plus Plater plate
  wrappers.
- Existing real/hybrid C++ APIs should be reused when correct.
- Placeholder and blocked tools must not appear enabled.
</truths>

<tasks>
1. Audit existing `EditorViewModel` object/volume/plate/gizmo invokables against
   `PREPWF-01..06`.
2. Add missing read-only gate properties or invokables needed by QML to decide
   whether each command is available.
3. Route object/plate operations through existing C++ service methods rather
   than adding QML-side business checks.
4. Add blocked/unsupported classification helpers for gizmo entries that depend
   on unavailable dependencies or absent backend implementations.
5. Ensure object/plate mutations invalidate slice result state where required.
6. Add focused `ViewModelSmokeTests` coverage for state propagation, object
   action gates, plate gates, and renderer-facing active plate membership.
</tasks>
