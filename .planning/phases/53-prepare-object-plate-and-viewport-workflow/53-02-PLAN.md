# Plan 53-02: Prepare QML object list, plate bar, viewport toolbar restoration

<objective>
Replace or repair the Prepare-page QML surfaces so object actions, plate actions,
view controls, and gizmo entries are screenshot-aligned and backed by the C++
contracts from Plan 53-01.
</objective>

<truths>
- QML is presentation and interaction wiring only.
- The object list must be a compact operational tree/list, not a decorative card
  stack.
- Viewport overlay controls must not block the central GLViewport interaction
  surface.
- Every enabled command must be backed by real or classified behavior.
</truths>

<tasks>
1. Repair or replace corrupted/comment-heavy blocks in touched Prepare QML with
   English ASCII source comments only.
2. Bind object list toolbar, row menu, inline rename, printable toggle, volume
   children, and destructive actions to C++ gate state.
3. Bind the plate bar and plate context menu to C++ plate availability, current
   plate, printable, lock, stale slice, and object-count state.
4. Rebuild `GLToolbars.qml` as a stable overlay component with compact icon
   buttons, tooltips, supported/blocked states, and no dead click handlers.
5. Ensure import and drag/drop flows update object list, plate bar, and viewport
   through existing `EditorViewModel` state.
6. Add source-level QML audits for command wiring, gate usage, and absence of
   enabled placeholders.
</tasks>

<files>
- `src/qml_gui/panels/ObjectList.qml`
- `src/qml_gui/components/GLToolbars.qml`
- `src/qml_gui/pages/PreparePage.qml`
- `tests/QmlUiAuditTests.cpp`
</files>

<verification>
- QML source audit confirms toolbar/menu commands are wired to C++ state.
- No empty handlers are introduced.
- Unsupported gizmo entries are not enabled without a block reason.
- Manual visual path remains: import, select, plate switch, camera rotate, tool
  switch.
</verification>
