# Phase 97 Plan: Thumbnail Save-Reload Round-Trip

## Objective

Close the v3.2 THUMB-02 loop: prove that a thumbnail persisted on save survives a
save → reload round-trip, and lock that contract with an automated test.

Phase 96 closed the WRITE side (`buildPlateDataList` populates
`PlateData::plate_thumbnail` at `ProjectServiceMock.cpp:5138-5139`;
`saveProject` populates `StoreParams::thumbnail_data` at `:5189-5198`). The READ
side has been complete since v3.2 Phase 30 (`ProjectServiceMock.cpp:5518-5529`
extracts `plate->plate_thumbnail` → `QImage(Format_RGBA8888)` into
`pendingPlateThumbnails_`, applied at `:5721-5723` via `setThumbnail`). Phase 97
is the end-to-end verification that the two halves actually meet:

1. **THUMBRT-01 (round-trip restores pixels):** saving a project with a captured
   thumbnail and reloading it restores the thumbnail via the existing read side so
   the reloaded `PartPlate::thumbnail()` matches the saved pixels. The format
   symmetry (`convertToFormat(Format_RGBA8888)` write ↔ `Format_RGBA8888` read at
   `:5526`) plus lossless tdefl PNG encoding makes an exact pixel match
   achievable.

2. **THUMBRT-02 (automated test):** an automated round-trip test saves a project
   with a KNOWN synthesized `QImage` (NOT an RHI capture — this isolates
   persistence from capture, per the CONTEXT test design), reloads it via a fresh
   `ProjectServiceMock`, and asserts the reloaded `PartPlate::thumbnail()`
   survives.

The test uses a known `QImage` directly (set via the `plateListMut()` test seam)
rather than the RHI capture path (Phase 95), so Phase 97 tests PERSISTENCE in
isolation from CAPTURE. Phase 95 (capture) and Phase 97 (persistence) are thus
verified independently.

Phase 97 is primarily a test phase. If the round-trip exposes a gap in the
already-reported-complete read side, task 97-01-02 applies the MINIMAL wiring
fix (the CONTEXT explicitly allows this contingency). Mock-generator removal,
final canonical verifier run, and runtime capture evidence are Phase 98.

## Requirements

- `THUMBRT-01`: Saving a project with captured thumbnails and reloading it
  restores the thumbnails via the existing read side
  (`ProjectServiceMock.cpp:5518-5529` → applied at `:5721-5723`) so the reloaded
  `PartPlate::thumbnail()` matches the saved pixels (THUMB-02 closure).
- `THUMBRT-02`: The round-trip is covered by an automated test (PartPlateTests or
  a new round-trip test) that saves a project with a known thumbnail, reloads it,
  and asserts the thumbnail pixels survive.
