# Phase 56 -- Parameter Settings Dialogs Restoration - UI Design Contract

> Visual and interaction contract for three independent settings dialogs (printer, material, process). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform:** Desktop QML / Qt 6.10 / C++17 (NOT web). Design system locked: Theme singleton + Cx* controls.
> **Visual truth frozen:** Phase 50 inventory (50-INVENTORY.md) with immutable region IDs SETPRINT-* and SETMAT-*.

| `Theme.spacingXS` | 4px | Inline icon-label gaps, dirty dot margin, badge padding |

## Dialog Shell Contract

### Window Structure (SETPRINT-SHELL / SETMAT-SHELL / SETPROC-SHELL)

Each settings dialog is an independent non-modal `ApplicationWindow` (not `CxDialog` which is modal-only). Three instances, parameterized by preset category.

| Property | Value | Source |
|----------|-------|--------|
| Type | `ApplicationWindow` | CONTEXT.md decision: non-modal, separately focusable |
| Modality | `Qt.NonModal` | Upstream Tab.cpp is non-modal floating panel |
| Default size | 920 x 640px | Screenshot density analysis |
| Minimum size | 720 x 480px | Prevent unusable small window |
| Title | Category-specific (see copywriting) | Upstream Tab title pattern |
| Frame color | `Theme.bgElevated` (`#2a3140`) | Per screenshot: dark dialog frame |
| Title bar height | `Theme.titleBarHeight` (40px) | Theme token |

### Title Bar Pattern

Custom title bar (not native window decorations -- per existing CxDialog header pattern):

```
[Icon/Preset Name]  [Preset Dropdown: CxComboBox] .............. [Modified Badge] [Save As] [X Close]
```

- Left: Category icon glyph + preset display name
- Center-left: `CxComboBox` for preset selection (drives `ConfigViewModel::setCurrentXxxPreset`)
- Right: dirty "Modified" badge (visible when `configVm.isPresetDirty`), Save As button (`CxButton` compact), Close button (matches CxDialog close pattern: 28x28px, "X" glyph, `Theme.chromeDangerHover` on hover)
- Title bar background: `Theme.chromeSurface` (`#10161e`)
- Title text color: `Theme.chromeText` (`#cdd7e6`)

### Close Behavior

Per CONTEXT.md decision:
- X button or Esc closes the dialog
- On close when dirty: show `UnsavedChangesDialog` (existing reusable dialog)
  - Save: calls `configVm.requestSavePendingChanges()`, then close
  - Discard: calls `configVm.requestDiscardPendingChanges()`, then close
  - Cancel: abort close, dialog stays open

### Open Path

Per CONTEXT.md: opened from Prepare left-sidebar "Setting" button per section.
- BackendContext forwards `settingsRequested(category)` signal to open correct dialog
- Dialog scoped to active preset category (printer/filament/process)
- Each dialog instance is created once and shown/hidden (not destroyed per open)

## Scroll / Tabs / Group-Nav Behavior

### Tab Switching

- Tabs filter which groups appear in the left group nav (driven by `ConfigOptionModel::pageNames()` + `filterIndicesByPage()`)
- Tab switch does NOT trigger unsaved-changes dialog (tabs are within the same dialog/preset)
- Tab switch preserves scroll position per tab (or resets to top -- planner discretion)

### Group Navigation

- Left sidebar uses `CxScrollView` wrapping a `ColumnLayout` of group items
- Selecting a group scrolls the main option area to the first option in that group
- "All" item shows all groups merged (no group filtering)
- Active group highlighted per above spec

### Search Behavior (per-dialog, SETTINGS-06)

- Per CONTEXT.md: per-dialog search scope
- Search input in the search bar (not a separate popup -- the existing `SearchDialog.qml` popup remains available via the Search button for cross-category search)
- Search filters `configVm.filterOptionIndices(category, searchText)` in real-time
- Search text non-empty: show match count, highlight matching label portions (reuse `highlightParts()` pattern from existing SearchDialog.qml)
- No matches: show empty state `qsTr("No matching options")` centered in option area
- Basic/Advanced toggle via `CxSwitch` drives `advancedMode` parameter to `filterOptionIndices()`

### Main Content Scroll

- Option area uses `ListView` with `clip: true`
- Vertical `ScrollBar` visible when `contentHeight > height`
- Alternating row background for readability
- Group headers sticky or non-sticky (planner discretion; upstream does not use sticky headers)

### New Cx* Control Needed

**CxUnitSpinBox** (or extend CxSpinBox): A `SpinBox` variant that renders a unit suffix text label to the right of the input field (e.g. "mm", "%", "C", "s", "mm/s"). The unit string is a `property string unit: ""` that renders as `Text` anchored right of the spinbox content. This is needed because the upstream settings UI shows units inline, and the current `CxSpinBox` does not support suffix text. Flag for planner: either extend `CxSpinBox` with a `suffix` property, or create a small wrapper component.
