# Plan 30-03: 3MF thumbnail persistence (THUMB-02)

**Objective:** Wire thumbnail persistence through 3MF save (buildPlateDataList populates plate_thumbnail) and load (write extracted pixels back to PartPlate). Closes THUMB-02.

## Tasks

### 30-03-T1 — Populate PlateData::plate_thumbnail in buildPlateDataList

**read_first:** `src/core/services/ProjectServiceMock.cpp:4806+` (buildPlateDataList — current PlateData construction); `third_party/OrcaSlicer/src/libslic3r/GCode/ThumbnailData.hpp` (ThumbnailData struct: width, height, pixels, set(w,h), is_valid); `src/core/model/PartPlate.h` (thumbnail() getter from Plan 30-01).

**action:** In buildPlateDataList, after constructing each `PlateData`, populate its `plate_thumbnail`:
1. Get the plate's QImage: `QImage img = plate->thumbnail();`
2. If `img.isNull()`, generate it: call the main-variant generator to produce a QImage (decode the base64 PNG via QImage::fromData), then `plate->setThumbnail(img)`.
3. Convert QImage to ThumbnailData:
   ```
   if (!img.isNull()) {
     QImage rgba = img.convertToFormat(QImage::Format_RGBA8888);
     pd.plate_thumbnail.set(rgba.width(), rgba.height());
     pd.plate_thumbnail.pixels.assign(rgba.bits(), rgba.bits() + rgba.sizeInBytes());
   }
   ```
4. Add `#include <libslic3r/GCode/ThumbnailData.hpp>` if not present (likely already via Model.hpp chain — verify).

**acceptance_criteria:**
- buildPlateDataList contains `pd.plate_thumbnail.set(` with the QImage RGBA conversion
- buildPlateDataList contains `pd.plate_thumbnail.pixels.assign(`
- buildPlateDataList regenerates the thumbnail via the generator if `plate->thumbnail()` is null
- compiles green

### 30-03-T2 — Write extracted thumbnail back to PartPlate on 3MF load

**read_first:** `src/core/services/ProjectServiceMock.cpp:5110+` (3MF load path — plate reconstruction where PlateData::plate_thumbnail.pixels is already extracted by bbs_3mf.cpp:1640); the rebuild lambda at ~5300-5360 where PartPlate objects are created and configured.

**action:** In the 3MF load rebuild lambda, after creating each PartPlate from the loaded PlateData, write the extracted thumbnail back:
1. For each loaded plate with `plate_data->plate_thumbnail.is_valid()`, convert its pixels back to QImage and set on the PartPlate:
   ```
   if (plate_data->plate_thumbnail.is_valid()) {
     QImage img(reinterpret_cast<const uchar*>(plate_data->plate_thumbnail.pixels.data()),
                plate_data->plate_thumbnail.width, plate_data->plate_thumbnail.height,
                plate_data->plate_thumbnail.width * 4, QImage::Format_RGBA8888);
     plate->setThumbnail(img.copy());  // copy() detaches from the buffer
   }
   ```
2. This ensures the thumbnail survives save→reload (the round-trip).

**acceptance_criteria:**
- The 3MF load rebuild lambda contains `setThumbnail(img.copy())` or equivalent, guarded by `plate_thumbnail.is_valid()`
- compiles green
