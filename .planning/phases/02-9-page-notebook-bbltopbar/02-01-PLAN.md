<objective>
Add a TabPosition enum and `requestSelectTab(int)` signal/slot to `BackendContext`, exposing them to QML. This is the C++ foundation that Plan 02-02 (BBLTopbar.qml rewrite) consumes — QML will reference `backend.TabPosition.tp3DEditor` instead of hardcoded integer `1`, and tab clicks will go through `backend.requestSelectTab(pos)` (mirroring upstream `MainFrame::request_select_tab` posting `EVT_SELECT_TAB`).

Purpose: Eliminate value-desync risk between C++ and QML (Pitfall 1 in 02-RESEARCH.md) by giving the integer enum a single canonical source. Establish the upstream-aligned broadcast primitive that replaces the current ad-hoc `pendingSwitchToken` / `pendingSwitchTargetPage` local state.

Output: Modified `BackendContext.h` (Q_ENUM + signal + Q_INVOKABLE), `BackendContext.cpp` (impl), `tests/ViewModelSmokeTests.cpp` (two new tests gated on BackendContext-only, NOT on HAS_LIBSLIC3R).
</objective>
