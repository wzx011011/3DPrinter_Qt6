# Phase 120 Plan: TriangleSelector Engine Port

## Objective
Close PAINT-01: port the TriangleSelector triangle-pick + adaptive-subdivide + paint-state pipeline so mouse input resolves to a triangle index and paint states apply with subdivision. TriangleSelector (libslic3r) is ALREADY compiled in — this phase builds the Qt6 owner/wrapper + bridges the 3 structural gaps. Foundation for Phase 121-123.

  <truth id="TS-06">RhiViewport gains an emitPaintPickIfActive(position, modifiers) parallel to emitMeasurePickIfActive (RhiViewport.cpp:929-959): stage-1 picks the source object, builds the world ray, emits to EditorViewModel.paintAtFacet. Gated on gizmoMode in {GizmoSupportPaint, GizmoSeamPaint, GizmoMmuSegmentation}.</truth>
  <truth id="TS-07">A `triangleSelectorEnginePorted` source-audit slot asserts: (a) ProjectServiceMock has volumeMeshTriangleMesh; (b) SceneRaycasterHit has meshLocalPosition; (c) PaintEngine.{h,cpp} exist and reference Slic3r::TriangleSelector + select_patch; (d) EditorViewModel references PaintEngine; (e) no reimplementation of TriangleSelector (assert PaintEngine.cpp includes TriangleSelector.hpp, not a hand-rolled selector).</truth>
  <truth id="TS-08">A pure helper for cursor-construction + select_patch invocation is extracted so it is unit-testable without a Model/renderer (mirrors v4.5/v4.6 pattern).   <truth id="TS-09">Canonical build (j6) + regression ctest passes; encoding guard clean.</truth>
