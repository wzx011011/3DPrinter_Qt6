# Plan 86-01: Restore Settings Option Sections And Typed Rows

## Goal

Restore screenshot/source-truth option section headers, typed editor visuals,
unit/range presentation, and compact state indicators without breaking existing
`ConfigOptionModel` and `ConfigViewModel` edit semantics.

## Must Haves

- D-01: keep Phase 85 settings shell intact and limit visual
  restoration to the option-content region.
- D-02: section headers must use a compact cyan icon/glyph
  rail, bold section title, and divider line, with no nested cards.
- D-03: sections must remain driven by existing
  `ConfigOptionModel::optGroup()` and filtered row order.
- D-04: row heights must remain stable through delegate
  `OptionRow.totalHeight`.
- D-05: every editable control must call
  `optionModel.setValue(root.optIdx, value)`.
- D-06: bool, numeric, enum, string, and color-like rows must
  use project `Cx*` controls or audited compact editor frames.
- D-07: units must prefer `optSidetext()` and fall back to
  `optUnit()`.
- D-08: range/min-max visuals must be present for range-like
  keys using existing metadata.
- D-09: color-like string rows must expose a swatch/button
  while preserving the current string value path.
- D-10: dirty, value-source, read-only, nullable, vector, and
  validation metadata must use a fixed metadata lane.
- D-11: read-only rows keep visible labels, units, and values.
- D-12: numeric bound validation visibility must use min/max
  metadata and stay non-jumping.
- D-13: add RED-first audit coverage before production QML edits.
- D-14: run targeted QML and viewmodel tests before final canonical verify.
- D-15: run encoding guard for touched files before committing.
