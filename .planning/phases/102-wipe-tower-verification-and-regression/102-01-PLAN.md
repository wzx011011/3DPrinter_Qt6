# Phase 102 Plan: Wipe-Tower Verification And Regression

## Objective

Close the v4.4 milestone (Wipe-Tower Geometry Readback And Real Rendering)
by consolidating the 8 WT-* source-audit expectations from the Phase 99 gap
matrix into a regression-lock test, running the canonical verifier,
confirming OWzxSlicer.exe launch liveness, and confirming the regression
ctest passes — including the new wipe-tower slots from Phase 100/101.

Phase 99 audited 8 regions (WT-PLACEHOLDER-BOX through WT-SOFTWARE-VIEWPORT)
and froze 3 decisions. Phases 100-101 implemented the readback (WTREAD-01/02)
and the Option A rendering baseline (WTRENDER-01/02). Phase 102 is the
end-to-end verification that closes the loop:

1. **WTVERIFY-01 (source-audit consolidation):** Add a QmlUiAuditTests slot
   `wipeTowerReadbackAndRenderAnchorsPresent` that asserts the WT-* source
   anchors are present across the codebase — a regression lock preventing
   future removal of the readback wiring, the PreparePage binding, the Option
   A baseline comment, or the SoftwareViewport box. The 8 regions map to
   concrete grep targets (see must_haves).

2. **WTVERIFY-02 (canonical verifier + launch + regression ctest):** Run
   `scripts/auto_verify_with_vcvars.ps1`, confirm `OWzxSlicer.exe` launches
   (5-second liveness per STATE.md deferred-items bar — Windows capture API
   blocked, so reachability via process-liveness + ctest + verifier, not
   screenshot), and confirm regression ctest 4/4 pass.

Phase 102 is a verification + regression-lock phase. It does NOT implement
new behavior — the substantive work shipped in Phases 99-101.
