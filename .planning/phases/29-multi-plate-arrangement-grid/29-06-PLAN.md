# Plan 29-06: Load-path defensive updatePlateOrigins + QmlUiAudit regression (LOW PRIORITY)

**Objective:** Add a defensive `updatePlateOrigins()` call at the end of the 3MF load-path rebuild lambda (ProjectServiceMock.cpp:5319-5328) per RESEARCH section 6. This guarantees plate origins are consistent even if the load-path auto-arrange (ProjectServiceMock.cpp:5334) is later made conditional (e.g., user disables auto-arrange-on-load). Also confirm QmlUiAuditTests stays green (no new QML in Phase 29).

**Note:** This plan is LOW priority (opportunistic). The load-path auto-arrange at line 5334 will already trigger rebuildPlatesAfterArrangement (via Plan 04's rewired arrangeObjects) → updatePlateOrigins, so origins are already correct post-load. This plan adds belt-and-suspenders. It can be skipped if time-constrained without breaking ARRANGE-01/02/03.

```
// Defensive origin recompute (RESEARCH §6): guarantees origins are consistent
```
