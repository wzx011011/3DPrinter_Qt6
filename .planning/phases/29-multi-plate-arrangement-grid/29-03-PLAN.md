# Plan 29-03: rebuildPlatesAfterArrangement on PartPlateList (ARRANGE-02 domain)

**Objective:** Implement `PartPlateList::rebuildPlatesAfterArrangement(bool exceptLocked, bool recyclePlates)` — the ARRANGE-02 domain logic that reconstructs per-plate membership from post-arrange translations via computePlateIndex. This is the core of Phase 29: it closes the gap where arrangeObjects called `Slic3r::arrange_objects` but never translated the result back into PartPlate membership (objects stayed on plate 0). Uses computePlateIndex on model instance translations because bed_idx is reset to 0 by ModelArrange.cpp:98 (RESEARCH finding #5 — the architectural reason for Phase 29).

### 29-03-T2 — Implement rebuildPlatesAfterArrangement

**read_first:** `src/core/model/PartPlateList.cpp` (current state after Plan 01; createPlate/deletePlate/reindex helpers); `src/core/model/PartPlate.h` (clearInstances at 156, addInstance at 143, removeInstance at 148, empty() from Plan 01, objToInstanceSet at 138, isLocked at 111); `third_party/OrcaSlicer/src/slic3r/GUI/PartPlate.cpp:6096-6139` (rebuild_plates_after_arrangement full body); `:6111-6127` (recycle loop); 29-RESEARCH.md section 2.1 (Qt6 port shape steps 1-3), 2.3 (bed_idx reset rationale), 8 gotchas 4/5 (tolerantVfn partial-apply note).

**action:** Implement `void PartPlateList::rebuildPlatesAfterArrangement(bool exceptLocked, bool recyclePlates)` under `#ifdef HAS_LIBSLIC3R` in PartPlateList.cpp:
1. Early-out guard: `if (!m_model) return;`
2. CLEAR non-locked memberships: `for (int i = 0; i < plateCount(); ++i) { if (!(exceptLocked && plate(i)->isLocked())) plate(i)->clearInstances(); }` — locked plates' instance sets preserved when exceptLocked=true (mirror upstream `clear(false,false,except_locked,...)` at PartPlate.cpp:5261).
3. RE-DISTRIBUTE via computePlateIndex: iterate model objects + instances:
   ```
   for (int objIdx = 0; objIdx < int(m_model->objects.size()); ++objIdx) {
     const auto& obj = m_model->objects[objIdx];
     for (int instIdx = 0; instIdx < int(obj->instances.size()); ++instIdx) {
       Slic3r::Vec3d offset = obj->instances[instIdx]->get_offset();  // world translation in mm
       int plateIdx = computePlateIndex(offset.x(), offset.y());
       if (plateIdx < 0) plateIdx = 0;  // clamp
       // Create plates up to plateIdx if needed (up to kMaxPlateCount)
       while (plateCount() <= plateIdx && plateCount() < kMaxPlateCount) createPlate();
       if (plateIdx < plateCount()) plate(plateIdx)->addInstance(objIdx, instIdx);
     }
   }
   ```
   Note: createPlate (Plan 01) calls updatePlateCols+updatePlateOrigins, so cols stays consistent as plates are added. The sort-by-arrange_order from upstream (PartPlate.cpp:6103) is omitted — Qt6's computePlateIndex decodes plate membership directly from translation, so iteration order does not affect the result. Add a comment: `// Upstream sorts by arrange_order before reload; Qt6 decodes plate index from each instance's translation directly, so ordering is irrelevant (D-29-5 divergence from PartPlate.cpp:6103).`
4. RECYCLE trailing empty non-locked plates (recyclePlates=true) — mirror PartPlate.cpp:6111-6127 EXACTLY:
   ```
   if (recyclePlates) {
     for (int i = plateCount() - 1; i > 0; --i) {
       if (plate(i)->empty()) {
         deletePlate(i);  // deletePlate guards keep>=1; Plan 01 wires reindex+updatePlateOrigins
       } else if (plate(i)->isLocked()) {
         continue;
       } else {
         break;  // contiguous trailing run only
       }
     }
   }
   ```
   Uses empty() from Plan 01 (NOT has_printable_instances — RESEARCH gotcha 6 defers it; needs model backref on PartPlate which Qt6 doesn't have). The reverse iteration stops at i==1 so plate 0 is NEVER deleted.
5. Call `updatePlateOrigins();` at the end so all surviving plates have correct origins for their new grid positions.
6. Add a non-HAS_LIBSLIC3R stub: `#else void PartPlateList::rebuildPlatesAfterArrangement(bool, bool) {} #endif`.

**acceptance_criteria:**
- PartPlateList.cpp contains `void PartPlateList::rebuildPlatesAfterArrangement(bool exceptLocked, bool recyclePlates)` with the HAS_LIBSLIC3R guard
- The clear step uses `plate(i)->clearInstances()` and skips locked plates when `exceptLocked` is true (`if (!(exceptLocked && plate(i)->isLocked()))`)
- The distribute step calls `computePlateIndex(offset.x(), offset.y())` on each instance's `get_offset()`
- The distribute step creates plates via `createPlate()` when `plateCount() <= plateIdx` up to `kMaxPlateCount`
- The recycle loop iterates `for (int i = plateCount() - 1; i > 0; --i)` (stops at i==1, never deletes plate 0)
- The recycle loop calls `plate(i)->empty()` (from Plan 01) and `plate(i)->isLocked()`
- The recycle loop calls `deletePlate(i)` for empty, `continue` for locked, `break` for first non-empty-non-locked
- A `#else` stub exists for non-HAS_LIBSLIC3R builds
