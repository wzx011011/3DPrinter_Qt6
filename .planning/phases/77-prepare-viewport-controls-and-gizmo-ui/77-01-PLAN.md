# Plan 77-01: Viewport Controls And Gizmo UI

## Goal

Restore Prepare viewport controls and gizmo floating UI toward screenshot
parity while preserving existing RHI gizmo behavior and C++ ViewModel gates.

## Tasks

1. RED static audit
   - Add a focused `QmlUiAuditTests` method for Phase 77.
   - Assert icon-first toolbar tokens and absence of text-only viewport tool
     labels.
   - Assert lower-left view controls and shared gizmo panel positioning tokens.
   - Run the focused source audit and capture expected failures.

2. Toolbar restoration
   - Replace `GLToolbars.qml` text buttons with icon-first controls.
   - Center the main action toolbar at the top of the viewport.
   - Move the gizmo tool stack to the right-side workspace area.
   - Move view preset/fit controls to a lower-left compact cluster.
   - Preserve `availableGizmoMask`, `canAddPlate`, and camera API bindings.

3. Gizmo floating panel restoration
   - Add compact transform mini-panel for move, rotate, and scale modes.
   - Normalize existing floating panels to shared compact top offset and smaller
     radius where this can be done without changing behavior.
   - Keep panel visibility tied to `viewport3d.gizmoMode` and existing
     ViewModel properties.

4. Verification
   - Run the focused source audit.
   - Run `git diff --check`.
   - Run encoding guard.
   - Run the canonical verifier if source/QML changes pass.
   - Launch `build/OWzxSlicer.exe` and record whether the window responds.

5. Closeout
   - Write summary, verification, code review, and UI review artifacts.
   - Update ROADMAP/STATE/REQUIREMENTS if checks pass.

## Acceptance Criteria

- `VIEWUI-01`: viewport controls use screenshot-like placement, compact sizes,
  icon-first visual language, and stable disabled states.
- `GIZMOUI-01`: move/rotate/scale/cut-related panels are visible in correct
  modes, compactly positioned, and do not disconnect existing RHI interaction.
