# Phase 113 Plan: Scene And Mesh Raycaster Port

## Objective
Close MEASURE-02: port ray-triangle intersection from upstream MeshRaycaster into Qt6 pure-CPU helpers, with a two-stage pick (existing ObjectPicking AABB → new per-triangle ITS raycast) for mouse-move perf.

## Must Haves
<must_haves>
  <truth id="MR-01">A new `MeshRaycaster` class in `src/core/rendering/MeshRaycaster.h/.cpp` (pure CPU, no GL/wxWidgets deps) wraps a `shared_ptr<const indexed_triangle_set>` + a transform, and exposes `rayCast(const Vec3d& rayOrigin, const Vec3d& rayDir)` returning a hit result (position + normal + triangle index) or no-hit. The intersection math mirrors upstream MeshUtils.hpp:159+ (Möller–Trumbore or whatever upstream uses — cite the anchor).</truth>
  <truth id="MR-02">A `SceneRaycaster` Qt6 wrapper (new `src/core/rendering/SceneRaycaster.h/.cpp` OR extension of ObjectPicking) holds per-volume MeshRaycaster instances keyed by object+volume index, populated lazily from the Phase 112 `volumeMeshIts` accessor. Caches the raycasters (don't reconstruct per mouse-move); invalidates on model change. Provides `hitTest(rayOrigin, rayDir, candidateObjects)` that runs the per-triangle raycast on each candidate's volumes and returns the closest hit.</truth>
  <truth id="MR-03">The two-stage pick is wired: stage 1 = existing `ObjectPicking::pickSourceObject` (coarse AABB — narrows to candidate objects); stage 2 = new `SceneRaycaster::hitTest` (per-triangle ITS raycast on the candidate volumes only). This addresses pitfall 7 (upstream loops all volumes per mouse-move — Qt6 must not).</truth>
  <truth id="MR-04">The hit result includes: object index, volume index, triangle index, world-space hit position, world-space normal. Sufficient for Phase 114 (Measuring consumes the hit) + Phase 115 (snap UX picks the feature).</truth>
  <truth id="MR-05">Both files are added to `src/core/rendering/CMakeLists.txt` (or wherever the rendering helpers are registered) so they compile/link into owzx_app_core.</truth>
  <truth id="MR-06">A regression test (PartPlateTests or a new tests/MeshRaycasterTests.cpp) asserts: (a) a known ray hits a known triangle (e.g. a unit cube); (b) a miss returns no-hit; (c) the closest-hit semantics (multiple triangles, nearest wins). Deterministic, no GPU/display needed.</truth>
  <truth id="MR-07">A QmlUiAuditTests source-audit slot `meshAndSceneRaycasterPorted` asserts MeshRaycaster + SceneRaycaster exist + the upstream anchor cite. MEASURE-02-named QVERIFY2 messages.</truth>
  <truth id="MR-08">Canonical build + regression ctest passes; OWzxSlicer.exe links clean.</truth>
  <truth id="MR-09">`git diff --check` exit 0; encoding guard clean.</truth>
</must_haves>

<threat_model>
Pure-CPU math helpers. The ITS input is trusted (from Phase 112 accessor, geometry data). Ray-triangle intersection is well-defined math; no parsing of external input. No network, no auth. Threats open: 0.
</threat_model>
