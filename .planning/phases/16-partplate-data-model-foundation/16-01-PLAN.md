# Plan 16-01: PartPlate + PartPlateList Domain Model

<objective>
Create the new `src/core/model/` domain layer with `PartPlate` (a Qt6-native value object mirroring upstream data/lifecycle/IO fields, instance-level membership, native DynamicPrintConfig) and `PartPlateList` (the container that owns plates and is the single source of truth). Back the new classes with deterministic unit tests BEFORE any ProjectServiceMock migration. This plan is non-destructive — it adds new files without touching existing plate behavior (that is plan 16-02).

This is the foundation every later v3.0 phase builds on, so it must be correct and independently testable before the big-bang migration.
</objective>

<must_haves>
  <truth>src/core/model/PartPlate.h defines a class mirroring upstream PartPlate.hpp:77-557 data/lifecycle/IO subset</truth>
  <truth>PartPlate holds std::set<std::pair<int,int>> instance membership (obj_to_instance_set equivalent)</truth>
  <truth>PartPlate holds Slic3r::DynamicPrintConfig m_config under #ifdef HAS_LIBSLIC3R (not QHash)</truth>
  <truth>PartPlate EXCLUDES all GL/wx fields (no GLModel/PickingModel/GLTexture/wxCoord)</truth>
  <truth>src/core/model/PartPlateList.h owns the plate collection + current plate index</truth>
  <truth>Unit tests in ViewModelSmokeTests.cpp exercise the new model and pass</truth>
  <truth>CMakeLists.txt compiles the new model files into owzx_app_core</truth>
</must_haves>
