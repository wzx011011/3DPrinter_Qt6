# Plan 26-01: Preview Segment Pipeline + GCV1 Parsing + Layer-Range Draw Control

<objective>
Add the CanvasPreview render branch to RhiViewportRenderer: parse the GCV1 blob into a GPU segment buffer (Lines topology), build a layer-offset index for draw-range layer filtering (eliminating the per-frame CPU filter), and handle visibility toggles (travel/extrusion). This is the core Preview rendering — color modes are pre-baked by PreviewViewModel (D-26-03), so the renderer just consumes opaque RGBA vertices.

**Scout finding:** RhiViewport already has all Preview Q_PROPERTYs; synchronize() already reads m_previewBytes (but discards the data); the m_linePipeline + camera-uniform SRB + rhi_viewport shaders already exist and match the segment vertex layout. This is primarily consuming existing infrastructure.
</objective>

<must_haves>
  <truth>RhiViewportRenderer::synchronize() stores preview data + control props (layerMin/Max/moveEnd/showTravelMoves) from RhiViewport</truth>
  <truth>RhiViewportRenderer::render() has a CanvasPreview branch that draws segments</truth>
  <truth>GCV1 blob is parsed into Line vertices (each PackedSegment → 2 vertices, axis-swapped)</truth>
  <truth>Layer-range filtering uses GPU draw range (firstVertex/vertexCount from a layer-offset index), NOT per-frame CPU re-filtering</truth>
  <truth>Travel-move visibility toggle is handled (show/hide travel segments without buffer re-upload)</truth>
  <truth>canonical verify exits 0; existing tests don't regress</truth>
</must_haves>

if (m_canvasType == RhiViewport::CanvasPreview && ensurePipelines()) {
