# Plan 24-01: Prepare Scene Data Contract And Dirty Flags

<objective>
Create the CPU-side Prepare scene/cache contract that later QRhi rendering consumes. This plan covers the testable part of RHI-04, PREP-01, and PREP-05 before GPU code changes: bed/plate state, generated bed/grid/origin geometry, active plate object membership, and explicit dirty flag consumption.

  <task id="24-01-T2" type="auto" tdd="true">
    <name>Implement the scene data and dirty flag contract</name>
    <files>
src/qml_gui/Renderer/PrepareSceneData.h
src/qml_gui/Renderer/PrepareSceneData.cpp
tests/PrepareSceneDataTests.cpp
    </files>
    <behavior>
Setting unchanged scene values must not mark extra dirty work. Changing bed metrics marks bed and GPU dirty. Changing current plate or active object indices marks plate and GPU dirty. Changing mesh byte generation marks mesh and GPU dirty. `takeDirtyFlags()` returns and clears accumulated dirty flags while `peekDirtyFlags()` observes without clearing.
    </behavior>
    <action>
Create a pure C++ helper colocated with the QRhi renderer boundary. Suggested API: small vertex structs for fill and line data, an integer dirty flag mask, `setBed(...)`, `setShowBed(bool)`, `setPlateContext(int currentPlateIndex, int plateCount, const QList<int>& activeObjectIndices)`, `setMeshGeneration(qint64)`, `peekDirtyFlags()`, `takeDirtyFlags()`, and const accessors for bed fill, grid lines, border/origin lines, current plate index, plate count, and active object indices. Clamp or reject invalid bed dimensions so tests prove a bad config cannot allocate unbounded geometry.
    </action>
    <verify>
      <automated>build/PrepareSceneDataTests.exe</automated>
    </verify>
    <done>
The new scene data object compiles, tests pass, and the API is ready for `RhiViewportRenderer::synchronize()` without depending on QML script logic.
    </done>
  </task>

<threat_model>
| Threat | STRIDE | Mitigation | Disposition |
|--------|--------|------------|-------------|
| Invalid bed dimensions create huge CPU or GPU buffers | Denial of Service | Clamp or reject dimensions and unit-test bounded geometry counts | mitigate |
| Empty selected plate silently renders all objects | Information Disclosure | Active plate context keeps an empty object list for invalid or empty plates | mitigate |
| Dirty flags are cleared accidentally before QRhi upload | Tampering | Separate `peekDirtyFlags()` from `takeDirtyFlags()` and test consumption semantics | mitigate |
</threat_model>

<success_criteria>
- PREP-01 covered at the data level: bed dimensions, grid, border, and origin cues are available to the renderer.
- PREP-05 covered at the data level: active plate object context is isolated and testable.
- RHI-04 covered at the CPU cache level: dirty categories are explicit and consumable by later GPU upload code.
- No QML or QRhi renderer code owns scene conversion in this plan.
</success_criteria>
