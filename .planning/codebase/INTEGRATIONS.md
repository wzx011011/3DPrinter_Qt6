# External Integrations

**Analysis Date:** 2026-05-31

## Overview

This project is a source-truth migration of CrealityPrint (a 3D printer slicer) from C++/wxWidgets to C++/Qt6/QML. The upstream source lives in `third_party/CrealityPrint`. All external integration points described below reflect the upstream architecture; the Qt6 migration currently uses mock implementations for most network/cloud/device services, with the libslic3r backend compiled from upstream source providing real slicing, mesh processing, and model I/O functionality.

## APIs & External Services

**Creality Cloud (BBL Cloud):**
- Purpose: User authentication, device binding/unbinding, cloud preset sync, print task management, firmware updates, model mall, OSS file storage
- Upstream implementation: `third_party/CrealityPrint/src/slic3r/Utils/NetworkAgent.cpp` (wraps bambu_networking library)
- Upstream header: `third_party/CrealityPrint/src/slic3r/Utils/NetworkAgent.hpp`
- Protocol: bambu_networking SDK (proprietary), MQTT over TLS, HTTPS REST
- Key functions: `login`, `logout`, `bind`, `unbind`, `connect_server`, `send_message`, `start_print`, `start_local_print`, `start_send_gcode_to_sdcard`, `get_user_presets`, `get_setting_list`, `put_setting`, `get_printer_firmware`
- Qt6 migration status: Mock implementation in `src/core/services/CloudServiceMock.h` (simulates login/logout/bind/sync without real network calls)

**Creality Printer Communication (LAN):**
- Purpose: Local device discovery (SSDP), printer connection, print job control, real-time status monitoring, HMS health notifications
- Upstream implementation: `third_party/CrealityPrint/src/slic3r/GUI/DeviceManager.cpp`
- Upstream header: `third_party/CrealityPrint/src/slic3r/GUI/DeviceManager.hpp`
- Protocol: MQTT (Paho MQTT C++ library), local socket connection, SSDP discovery
- Upstream MQTT client: `third_party/CrealityPrint/src/slic3r/GUI/print_manage/App/mqtt_client.h`
- Key functions: `connect_printer`, `disconnect_printer`, `send_message_to_printer`, `start_discovery`, `start_subscribe`, `stop_subscribe`
- Qt6 migration status: Mock implementation in `src/core/services/DeviceServiceMock.h` (simulates devices, print jobs, HMS notifications without real MQTT/SSDP)

**HTTP Server (Local):**
- Purpose: Local HTTP endpoint for inter-process communication (wxWebView integration in upstream)
- Upstream implementation: `third_party/CrealityPrint/src/slic3r/GUI/HttpServer.cpp`
- Upstream header: `third_party/CrealityPrint/src/slic3r/GUI/HttpServer.hpp`
- Technology: Boost.Beast + Boost.Asio
- Port: 15888 (customized builds) or 13666 (standard)
- Qt6 migration status: Not migrated; not needed in QML-first architecture (Qt has native signal/slot communication)

## Data Storage

**Local Configuration:**
- QSettings (`QCoreApplication::organizationName` = "CrealityDemo", `QCoreApplication::applicationName` = "Print7Shell")
- Used for: config wizard completed state (`src/qml_gui/BackendContext.cpp`)
- Upstream uses: Boost.PropertyTree + JSON files for configuration persistence

**Project Files (3MF):**
- Format: 3MF (ISO 4290) - standard 3D manufacturing format
- Parsing: libslic3r `Format/` module (`third_party/CrealityPrint/src/libslic3r/Format/`)
- Includes: 3MF loading, model import/export, project save/load
- STEP/SVG import: via OpenCASCADE (OCCT) libraries

**Presets & Profiles:**
- Format: JSON and custom binary formats
- Upstream: `PresetBundle`, `PresetUpdater` in slic3r Utils
- Qt6 migration status: Mock implementation in `src/core/services/PresetServiceMock.h`

**G-Code Files:**
- Generated by libslic3r slicing engine
- Parsed and rendered in preview via custom GL renderer (`src/qml_gui/Renderer/GCodeRenderer`)

**File Storage:**
- Local filesystem only (no cloud storage integration in Qt6 migration)
- Upstream has OSS (Object Storage Service) upload via bambu_networking for cloud print tasks

**Caching:**
- None detected (upstream has some caching in NetworkAgent)

## Authentication & Identity

**Auth Provider:**
- Creality Cloud (proprietary, via bambu_networking SDK)
- Authentication method: Username/password login, OAuth-like token flow
- Session management: Login commands, user info retrieval, logout
- Qt6 migration status: Mock login/logout in `CloudServiceMock` with simulated async behavior

**Upstream Web Login:**
- wxWebView-based OAuth flow in upstream (`WebUserLoginDialog`, `WebGuideDialog`)
- Uses bambu_networking `build_login_cmd`, `build_login_info` for constructing login payloads
- Qt6 migration status: Not migrated (would use Qt WebEngine or system browser OAuth)

## Monitoring & Observability

**Error Tracking:**
- Windows Crash Handler: `src/core/debug/CrashHandlerWin.cpp` - installs Windows structured exception handler, writes crash dumps to `crash_dumps/` directory
- Early crash diagnostics: `src/core/debug/EarlyCrashDiag.cpp`
- Crash dump directory: `<app_dir>/crash_dumps/`
- Crash handler log: `build/crash_dumps/crash_stack.log`

**Logs:**
- Startup diagnostics: `<app_dir>/startup_diagnostics.log` (written by `src/qml_gui/main_qml.cpp`)
- QML warnings: Captured to startup log via QQmlEngine::warnings signal
- QML binding/connection tracing: Enabled via `QT_LOGGING_RULES=qt.qml.binding=true;qt.qml.connections=true`
- Latency instrumentation: Built into `BackendContext` with beginLatency/endLatency/recordLatency (percentile-95 tracking)
- QML warning capture: `scripts/capture_qml_warnings.ps1`

## CI/CD & Deployment

**Hosting:**
- GitHub repository (git-based)
- No cloud hosting for the application itself

**CI Pipeline:**
- GitHub Actions (`.github/workflows/`)

## Environment Configuration

**Required env vars (build time):**
- `Qt6_DIR` - Qt 6.10 installation path
- `CMAKE_PREFIX_PATH` - Include Qt 6.10 and pre-built deps paths

**Optional env vars:**
- `QML_DEBUG_LOG` - Enable verbose QML debug logging to file
- `SLIC3R_STATIC` - Build with static libraries (upstream)
- `DUMPTOOL_USER`, `DUMPTOOL_PASS`, `DUMPTOOL_HOST`, `DUMPTOOL_TO` - Crash dump tool credentials (upstream)
- `BUILD_ID` - Build identifier (upstream)

**Secrets location:**
- Not detected in this repository. Upstream may use bambu_networking SDK certificates.

**Upstream-only env vars (not used in Qt6 migration):**
- Various BAMBU_NETWORK_* error codes defined in `bambu_networking.hpp` (return codes, not env vars)

## Video & Camera Streaming

**RTSP Camera Streams:**
- Purpose: Live camera feed from Creality 3D printers during printing
- Upstream implementation: `third_party/CrealityPrint/src/video/RTSPDecoder.cpp`
- Technology: FFmpeg (avformat, avcodec, avutil, swresample, swscale, avfilter)
- Status: FFmpeg not found in build environment; RTSP blocked

**WebRTC Camera Streams:**
- Purpose: Alternative camera streaming via WebRTC
- Upstream implementation: `third_party/CrealityPrint/src/video/WebRTCDecoder.cpp`
- Technology: MetaRTC SDK (metartc6, metartccore6), usrsctp, srtp2, opus, speexdsp
- Status: Not integrated in Qt6 migration

**Qt6 migration status:**
- Mock implementation: `src/core/services/CameraServiceMock.h`
- Simulates stream status transitions, recording, timelapse without real video decoding
- Real implementation would require Qt Multimedia or FFmpeg integration
