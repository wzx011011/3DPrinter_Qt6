# Phase 105 Plan: D3D12 Debug Layer Wiring

## Objective
Close D3D12-01: wire the D3D12 debug layer behind an env flag so Phase 106 can triage the startup `0xc0000005` access violation. Infrastructure only — does NOT investigate the crash.

## Must Haves
<must_haves>
  <truth id="DL-01">A new env flag `OWZX_D3D12_DEBUG` (values "1"/"true"/"on"/"yes" → enabled) gates the D3D12 debug layer. Read via `qEnvironmentVariableIsSet` + `qgetenv` (the existing pattern at RhiBackendSelector.cpp:117-124).</truth>
  <truth id="DL-02">When the flag is set AND the selected backend is D3D12, `QRhiD3D12InitParams::enableDebugLayer` is set to `true` BEFORE `QRhi::create` in the `probeBackend` path (RhiBackendSelector.cpp:80). A startup-log line records the enablement.</truth>
  <truth id="DL-03">The live QQuickRhiItem render path is also covered: either (a) the env flag is forwarded to Qt's QSG RHI debug mechanism (`qputenv("QSG_RHI_DEBUG", "1")` in main_qml.cpp before QGuiApplication, OR set `D3D12_INFORMATION_LEVEL` / `DXGI_INFO_QUEUE` separately), OR (b) a documented comment explains that the probe-path enablement is sufficient for Phase 106's investigation because the crash is reproducible at the QRhi::create level. The executor determines which is correct by checking where the crash actually fires (probe or live render).</truth>
  <truth id="DL-04">The enablement is GATED on the env flag — when the flag is unset (the default), `enableDebugLayer` stays false (no perf hit, no release-build leak per Pitfall 5). The default OWzxSlicer.exe build is unchanged.</truth>
  <truth id="DL-05">A `d3d12DebugLayerWiredBehindEnvFlag` (or similar) source-audit slot in QmlUiAuditTests asserts: (a) the env-flag name `OWZX_D3D12_DEBUG` appears in RhiBackendSelector.cpp; (b) `enableDebugLayer` appears in RhiBackendSelector.cpp; (c) the QGuiApplication-before timing constraint is documented. Mirrors the Phase 102/103/104 source-audit pattern.</truth>
  <truth id="DL-06">Canonical build + regression ctest 4/4 passes; OWzxSlicer.exe links clean; default launch (no env flag) liveness confirmed (no debug-layer leak into default build).</truth>
  <truth id="DL-07">`git diff --check` exit 0; encoding guard clean; English ASCII comments.</truth>
</must_haves>

No new attack surface. The debug layer is gated behind an env flag the user must explicitly set. No external input, no network, no auth. Debug layer enables verbose GPU validation output (local stdout/debugger). Threats open: 0.
