# Phase 112 Plan: Per-Volume ITS Accessor And Mesh Cache

## Objective
Close MEASURE-01: add a per-volume ITS accessor + ownership contract on ProjectServiceMock. The cross-workstream dependency that unblocks Phase 113-115 (GLGizmoMeasure) AND the AssembleViewDataPool ModelObjectsClipper (v4.2 deferred).

## Must Haves
<must_haves>
  <truth id="MI-01">A per-volume ITS accessor exists on ProjectServiceMock: `std::shared_ptr<const Slic3r::indexed_triangle_set> volumeMeshIts(int objectIndex, int volumeIndex) const` (or equivalent). Returns the per-volume mesh's ITS. The accessor is the PUBLIC API the Phase 113 raycaster + Phase 114 Measuring + AssembleViewDataPool will consume.</truth>
  <truth id="MI-02">The ownership contract is documented in a comment at the accessor declaration: who owns the ITS (shared_ptr — shared between the cache and callers), when it's invalidated (model load/change/remove — the cache is cleared on those signals), how it's keyed (objectIndex+volumeIndex). The shared_ptr ensures the ITS outlives any single caller even if the underlying ModelVolume is removed mid-use.</truth>
  <truth id="MI-03">If the libslic3r `vol->mesh().its` is safe to share by reference (the TriangleMesh owns it for the ModelVolume's lifetime), the accessor returns a shared_ptr that does NOT copy (or documents the shallow-share). If a copy is needed for safety (e.g. to decouple from libslic3r lifetime), document that. The executor determines the cleanest approach.</truth>
  <truth id="MI-04">A mesh cache (optional but recommended if construction is non-trivial): keyed by (objectIndex, volumeIndex), invalidated on model change. If `vol->mesh().its` is already in-memory and the shared_ptr share is safe, the cache may be unnecessary — document the decision.</truth>
  <truth id="MI-05">The accessor returns null/empty for invalid indices (out-of-range object/volume) — defensive, no crash.</truth>
  <truth id="MI-06">SurfaceFeature boundary scrubbing (pitfall 6) is DOCUMENTED in the accessor comment: raw `void* volume` / `vector<int>* plane_indices` from libslic3r `SurfaceFeature` MUST NOT escape into Qt — Phase 113/114 enforce this when the raycaster + Measuring land. The accessor itself does NOT return SurfaceFeature (it returns the ITS only); the scrubbing is a Phase 113/114 concern, flagged here.</truth>
  <truth id="MI-07">A regression test (ViewModelSmokeTests or PartPlateTests) asserts: (a) the accessor returns a non-null ITS for a valid object+volume; (b) returns null for an invalid index; (c) the ITS has the expected vertex/triangle count (matches the loaded model). Mirrors the v4.4 readback test pattern.</truth>
  <truth id="MI-08">A QmlUiAuditTests source-audit slot `perVolumeItsAccessorPresent` asserts the accessor exists + the ownership-contract comment is present. MEASURE-01-named QVERIFY2 messages.</truth>
  <truth id="MI-09">Canonical build + regression ctest passes; OWzxSlicer.exe links clean.</truth>
  <truth id="MI-10">`git diff --check` exit 0; encoding guard clean.</truth>
</must_haves>

<threat_model>
The accessor reads trusted libslic3r state (ModelVolume::mesh().its). The ITS is geometry data, no external input. The shared_ptr ownership prevents UAF across the libslic3r→Qt boundary (pitfall 6). No network, no auth. Threats open: 0.
</threat_model>
