# Roadmap: v5.5 Build/Run Parity and Dependency Provenance

**Defined:** 2026-07-23
**Status:** Complete
**Phases:** 188-192

## Goal

Make the local build/run path explainable, reproducible, and comparable with
GitHub CI without changing product behavior. v5.5 closes the trust gap exposed
by a green CI run plus a local canonical-script failure: the application can be
built and launched locally, but the dependency/source path provenance and
failure classification are not yet documented tightly enough.

## Scope Rule

- Only the canonical build command is valid:
  `powershell -ExecutionPolicy Bypass -File scripts/auto_verify_with_vcvars.ps1`
- Only `build/` may be used as the build directory.
- Qt is a prebuilt dependency, not built from source.
- This milestone may adjust planning docs and the canonical verification script.
- This milestone must not change user-facing slicer behavior.
- This milestone must not patch upstream OrcaSlicer code.
- LAN/cloud/device/camera/printer-hardware workflows remain out of scope.

## Workstreams

- **DOC:** Make planning state match reality after v5.4 and define v5.5 entry.
- **PROV:** Document Qt, Orca upstream, and dependency-bundle provenance.
- **VERIFY:** Make local script output classify configure/build/test/run results.
- **RUN:** Keep app launch liveness explicit and repeatable.
- **GATE:** Compare CI and local evidence without treating environment-only
  differences as product regressions.
