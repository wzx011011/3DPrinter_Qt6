# Requirements: v5.6 Deferred Backlog Closure

**Defined:** 2026-07-24
**Core Value:** Dispose of all 11 v5.5 deferred items in one milestone without
reopening the removed LAN/device/cloud/printer-hardware scope, without embedding
Python, and without promoting D3D12 to the default backend.
**Status:** Active.

## DOC: Planning State

- [ ] **DOC-01** (Phase 193): Top-level planning files identify v5.6 as the
  active milestone, archive v5.5 as complete, and state the v5.5 handoff (the
  canonical build/run path is documented and reproducible). No top-level file
  still names v5.5 as active.

## UI: Component and Dialog Unification

- [ ] **UI-01** (Phase 194): The inline `NumericEdit`/`Badge` components in
  `OptionRow.qml` are promoted to `controls/CxNumericEdit.qml` and
  `controls/CxBadge.qml`; the parallel step-button idioms in `MoveSlider.qml`
  (`MoveStepButton`) and `PreviewLayerRail.qml` (`RailButton`) are unified into
  `controls/CxStepButton.qml`; all 3+ reference sites are migrated. A new
  `controls/CxBusyIndicator.qml` is added for later phases. No duplicated inline
  definitions remain.

- [ ] **UI-02** (Phase 195): The shortcut overview dialog is extracted from the
  inline `main.qml` Dialog into an independent `dialogs/KBShortcutsDialog.qml`,
  reorganized as a 5-group view (Global/Prepare/Toolbar/ObjectsList/Preview)
  aligned with upstream `KBShortcutsDialog.cpp`. Displayed shortcuts match the
  actual `Shortcut {}` bindings (missing bindings added, stale entries removed).

## FEAT: Feature Feedback and Depth

- [ ] **FEAT-01** (Phase 196): `EditorViewModel` exposes an `embossRunning`
  Q_PROPERTY (true on async start, false on added/failed); the Emboss panel in
  `PreparePage.qml` shows a spinner (via `CxBusyIndicator`) instead of the
  static placeholder text during async text generation. `SliceProgress.qml`
  binds to `sliceService.sliceState` and renders dedicated Cancelled/Error UI
  branches aligned with upstream `BackgroundSlicingProcess`/`ProgressIndicator`.

- [ ] **FEAT-02** (Phase 197): The inaccurate `.drc` term is corrected to
  `.stl`/`.3mf`/`.step` across the repository (source comments and planning
  docs). The upstream `resources/calib/*` tower models (temperature_tower, VFA,
  retraction_tower, SpeedTestStructure, pressure_advance, flowrate, etc.) are
  packaged into a Qt6 resource or runtime path. `CalibrationServiceMock` selects
  and loads the tower model matching the calib mode (replacing
  `cloneCurrentPlateModel()`) and applies per-mode config overrides
  (`spiral_mode`, `wall_loops`, aligned with upstream `CalibUtils.cpp`).

- [ ] **FEAT-03** (Phase 198): `ObjectList.qml` is deepened from a 2-level
  (object -> volume) tree toward the upstream `GUI_ObjectList.cpp` model,
  adding a layer level (aligned with `GUI_ObjectLayers.cpp`) and object/part
  settings sub-panel entries (aligned with `GUI_ObjectSettings.cpp`). This is
  new work; the previously deleted `AuxiliaryPage` (removed device scope) is
  NOT revived.

## DLG: Dialog Real Backend

  PresetUpdater are NOT built (deferred).

- [ ] **DLG-02** (Phase 200): `ConfigWizardDialog.qml` is rewritten to read

  wizard is positioned as a single-vendor (Creality) wizard. Multi-vendor
  selection and PresetUpdater are deferred.

- [ ] **DLG-03** (Phase 201): A new `AmsMaterialsViewModel` (C++) carries the
  hardcoded data currently in `AMSSettingsDialog.qml:22-37`
  (slotColors/Names/Materials/AutoSwap/remainingPct). The dialog rebinds to the
  ViewModel. The "add mapping" button (currently `enabled: false`) is enabled
  and wired. The data source remains mock; no real device/cloud connection is
  added (user decision 2026-07-24: architecture cleanup only).

- [ ] **DLG-04** (Phase 202): A new `PluginService` (C++) provides a local
  plugin registry (QSettings-persisted enabled/disabled state) plus
  download/install/uninstall (HTTP + local files).
  `PluginManagerDialog.qml` is upgraded from mock to bind the real service,
  aligned with upstream `WebDownPluginDlg` (a wx C++ plugin download dialog).
  CPython is NOT embedded (user decision 2026-07-24).
