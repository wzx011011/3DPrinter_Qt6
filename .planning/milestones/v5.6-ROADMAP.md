# Roadmap: v5.6 Deferred Backlog Closure

## Goal

Close the v5.5 deferred backlog. Dispose of all 11 deferred items in a single
milestone: implement the 8 offline-deliverable items and document the remaining
3 per user decisions (keep deferred / architecture-only / root-cause only).

**Core Value:** clear the deferred list without reopening the
LAN/device/cloud/printer-hardware scope, without embedding Python, and without
promoting D3D12 to the default backend.

## Scope Rule

- **Allowed:** UI component unification, dialog deepening, calibration tower
  geometry, object-tree enhancement, Emboss/SliceProgress feedback, AMS
  mock-to-ViewModel architecture cleanup, plugin-download UI real backend, i18n
  strings, D3D12 root-cause investigation.
- **Forbidden:** reopening printer-hardware/network data sources; embedding
  CPython; promoting D3D12 to the default backend; modifying libslic3r slicing
  algorithms; patching upstream OrcaSlicer source; creating alternate build
  directories or non-canonical build scripts.
- **Canonical build command:** `powershell -ExecutionPolicy Bypass -File scripts/auto_verify_with_vcvars.ps1`
- **Canonical build directory:** `build/`

## Workstreams

- **DOC** — planning state reconciliation
- **UI** — control/option-row/slider unification and shortcut dialog
- **FEAT** — Emboss spinner, SliceProgress states, calibration geometry, object tree
- **DLG** — ConfigWizard, AMS cleanup, plugin manager
- **RHI** — D3D12 root-cause confirmation
- **I18N** — de/fr/ja/ko long tail to >=85%
- **GATE** — cross-workstream regression and milestone audit

## Verification Gate

Each phase PLAN.md must contain a Verification section (v5.6 changes product
behavior, so verification is stricter than v5.5):

- [ ] Canonical build exits 0.
- [ ] Relevant ctest targets PASS (including new enumeration/control/dialog
  tests).
- [ ] E2E workflow PASS.
- [ ] `OWzxSlicer.exe` launch liveness (PID reported).
- [ ] Regression slots intact (`v56CrossWorkstreamRegressionLocked` plus
  historical v5.4/v5.5 anchors).

Milestone close (Phase 205): canonical verifier green + audit status `tech_debt`
or `clean`.

## Risks

1. **D3D12 (Phase 203) may not conclude** — 0xC0000005 does not reproduce
   headless; requires the original machine plus debugger. If unavailable, the
   deliverable is a hypothesis report and the default stays D3D11 (not a
   milestone blocker).
2. **ObjectList tree (Phase 198) scope interpretation** — "Auxiliary file-tree
   panel" was underspecified; interpreted as ObjectList tree deepening. Clarify
   before Phase 198 if another reading was intended.
3. **Submodule not checked out** — upstream comparison uses `D:/work/OrcaSlicer`
   (v5.5 follow-up). Phase 197 calibration models can be copied directly from
   that path (not a hard blocker); Phases 198/199 upstream depth comparison is
   affected.
4. **i18n translation quality** — batch script translation needs human spot
   checks for terminology consistency (technical terms in particular).
