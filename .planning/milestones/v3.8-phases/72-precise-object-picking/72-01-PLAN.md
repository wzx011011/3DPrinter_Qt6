  <action>
    Create `ObjectPicking` as a static-only helper with:
    `static int pickSourceObject(const QVector3D &rayOrigin,
                                 const QVector3D &rayDirection,
                                 const QList<PrepareSceneData::ModelVertex> &vertices,
                                 const QList<PrepareSceneData::ModelBatch> &batches);`

    Implement ray-AABB prefiltering against each batch's `ModelBounds`.
    Parallel rays must miss only when the ray origin is outside that slab.
    Return the first positive box interval distance through an output parameter
    so callers can skip triangle scans for boxes behind an existing nearer
    triangle hit.

    Implement Moller-Trumbore ray-triangle intersection against each batch's
    `firstVertex..firstVertex+vertexCount` triangle stream. Use GL path
    tolerances: parallel epsilon `1e-6f`, hit distance threshold `1e-4f`.
    Reject invalid spans, source indices below zero, non-finite rays/vertices,
    and degenerate triangles. Track nearest positive `t` and return that
    batch's source object index.
  </action>
  <acceptance_criteria>
    - `ObjectPickingTests` precise picking tests pass.
    - Helper has no QRhi, QQuickItem, OpenGL, service, or viewmodel dependency.
    - Helper consumes scene-space `PrepareSceneData` vertices directly.
  </acceptance_criteria>
  <done>Precise picking is available as reusable tested CPU logic.</done>
