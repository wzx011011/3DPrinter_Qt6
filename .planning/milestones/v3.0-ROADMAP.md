## Overview

v3.0 replaced the mock plate shell (`int plateCount_` + 9 parallel `QList` vectors) with a real `PartPlate`/`PartPlateList` domain model, made multi-plate state round-trip through 3MF, honored the full per-plate config during slicing, and added the missing clone/reorder/printable lifecycle ops with QML UI. This fixed all 5 problems identified in the v2.9 gap analysis. The milestone shipped after both a code review (Phase 21, 2 P0 bugs fixed) and a UI review (Phase 22, 2 P1 findings fixed).

## Key Decisions

| Decision | Rationale | Outcome |
|---|---|---|
| New src/core/model/ layer with pure domain objects | Separate plate semantics from Qt adaptation; independently testable | Good — clean layering |
| Instance-level membership (std::set<pair<int,int>>) | Complete source-truth alignment with upstream obj_to_instance_set | Good — future-proof for multi-instance |
| Native DynamicPrintConfig config (not QHash) | Phase 18 3MF persistence + Phase 19 slice merge become native operations | Good — with QVariant adaptation view retained |
| Big-bang migration (delete 9 vectors, no bridge) | No deferred structural debt | Good — clean, but 8 build-fix cycles in Phase 18 |
| Stack-local Print = per-plate context (D-14) | Scout revealed Qt6 already isolates plates per-slice; upstream m_print_list is caching not correctness | Good — avoided a large refactor |
| buildPlateDataList as file-local free fn | Avoid libslic3r/Format/bbs_3mf.hpp header pollution | Good — pragmatic engineering |
| Round-trip test QSKIP (not QFAIL) | store_bbs_3mf needs valid geometry; honest fixture limitation | Honest — recorded as PLATE-09 partial + v3.1 priority |
