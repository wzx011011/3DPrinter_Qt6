<objective>
Lay the data + undo foundation for ASM-01: add per-instance assemble-transform
accessors to ProjectServiceMock that proxy to the upstream
`ModelInstance::m_assemble_transformation` fields under `HAS_LIBSLIC3R` (with
parallel mock stores for the non-lib path), and add an `AssembleTransformCommand`
undo command variant that writes the assemble transform instead of the ordinary
object transform. This unblocks Plan 02 (EditorViewModel routing) and Plan 04
(3MF round-trip — the accessors are what the round-trip test asserts against).

Upstream source truth: the assemble transform lives on `ModelInstance`
(Model.hpp:1253-1298) and is what the Move/Rotate/Scale gizmos write when on the
assembly canvas. The ordinary `m_transformation` (offset/rotation/scaling_factor)
is the Prepare/View3D target and stays untouched by this plan.
</objective>

<truths>
- T-01: Upstream stores the assembly move/rotate/scale on `ModelInstance::m_assemble_transformation` (Geometry::Transformation), NOT on `m_transformation`. See Model.hpp:1253-1255.
- T-02: The upstream accessors are `get_assemble_offset`/`set_assemble_offset`, `get_assemble_transformation`/`set_assemble_transformation`, `set_assemble_rotation`, plus `m_assemble_transformation.set_scaling_factor()` for scale. `set_assemble_offset`/`set_assemble_rotation` flip `m_assemble_initialized=true`. See Model.hpp:1280-1294.
- T-03: The 3MF `<assemble>` block serializes `get_assemble_transformation().get_matrix()` + `get_offset_to_assembly()` on save (only when `is_assemble_initialized()`), and reloads via `set_assemble_from_transform` + `set_offset_to_assembly` (bbs_3mf.cpp:8070-8088 write, 4734-4741 read). No new 3MF code is needed in this plan — the contract already exists.
- T-04: The existing per-OBJECT accessors (`objectPosition/Rotation/Scale`) follow the dual-path convention: proxy to `ModelInstance::get_offset/get_rotation/get_scaling_factor` under `HAS_LIBSLIC3R`, else fall back to `QList<QVector3D>` mock stores. The new assemble accessors MUST mirror this exact convention. See ProjectServiceMock.cpp:4916-4968.
- T-05: The slic3r<->GL axis convention for the ordinary offset is `slic3r(X,Y,Z) <-> GL(X,Z,Y)` (swap Y/Z). The assemble offset accessor MUST use the same swap so the gizmo world-delta (GL space) lands in the correct slic3r field. See ProjectServiceMock.cpp:4926-4927, 4983-4984.
- T-06: Rotation in Qt is degrees; upstream stores radians. The ordinary rotation accessor does `qRadiansToDegrees`/`qDegreesToRadians`. The assemble rotation accessor MUST match. See ProjectServiceMock.cpp:4943-4947, 5006-5008.
- T-07: Undo commands capture old/new state in the constructor + `setNewTransform`, then call the service setters on undo/redo. The existing `TransformCommand` writes the ordinary transform; an assemble-aware variant (or a flag on TransformCommand) is required so undo/redo on the assembly canvas restores the assemble transform, not the Prepare transform.
</truths>
