# Plan 136-01: CGAL MeshBoolean Compile/Link Enablement (compat-patch path)

**Phase:** 136 — CGAL 5.6+ Dependency Upgrade
**Status:** Complete (backfilled from commits `661f48c`, `a740147`)
**Requirement:** CGAL-01

## Objective

Make `MeshBoolean.cpp` compile and link in `libslic3r_cgal_from_source` WITHOUT upgrading the CGAL dependency bundle, by applying a 2-line compat patch for CGAL 5.4 and re-enabling the source in CMake.

## Tasks

1. **Submodule compat patch** (`a740147`) — advance `third_party/OrcaSlicer` to a commit where `src/libslic3r/MeshBoolean.cpp` has:
   - `#if 0` → `#if 1` on the corefine-based boolean path (CGAL 5.4 ships `corefinement.h`).
   - Output iterators added to match the 5.4 `corefine_and_compute_*` signature.
2. **CMake re-enable** (`661f48c`) — in `cmake/BuildLibslic3rFromSource.cmake`, add `MeshBoolean.cpp` back to the `libslic3r_cgal_from_source` sources list (it had been excluded under the false "needs CGAL 5.6+" assumption).
3. **Canonical build verification** — run `scripts/auto_verify_with_vcvars.ps1`; confirm clean link + all ctest groups PASS + app launch liveness.

## Verification

- `cmake/BuildLibslic3rFromSource.cmake` lists `MeshBoolean.cpp` in cgal sources.
- `third_party/OrcaSlicer` submodule HEAD contains the 2-line patch.
- Canonical build exit 0; 5/5 ctest PASS; `APP_RUNNING_PID` captured.
- No regressions in existing CGAL-dependent code (cut surface, mesh boolean prep).

## Risk

- **Low.** The compat patch is 2 lines in a third_party source file recorded via submodule advancement. If CGAL is ever upgraded to 5.6+, the patch becomes a no-op (the `#if 1` path is the canonical one) and can be dropped.

## Out of scope

- Flipping `kCgalMeshBooleanAvailable` to true and activating the boolean/drill call sites — that is Phase 137.
- Full 5.6+ dependency-bundle rebuild — deferred (see 136-CONTEXT.md).
