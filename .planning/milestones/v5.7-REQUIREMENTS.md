# Requirements: v5.7 D3D12 Default-Backend Promotion

**Defined:** 2026-07-24
**Core Value:** Promote D3D12 to the default QRhi backend on Windows behind an
automatic D3D11 fallback, after reproducing the historical crash as far as the
environment allows and mitigating the three candidate seams.
**Status:** Active.

## DOC: Planning State

- [ ] **DOC-01** (Phase 206): Top-level planning files identify v5.7 as the
  active milestone and archive v5.6 as complete. No top-level file still names
  v5.6 as active.

## REPRO: Reproduction and Diagnostics

- [ ] **REPRO-01** (Phase 207): (a) `RhiViewportRenderer` gains structured
  crash-time diagnostics that dump render state (canvas type, pipeline ready,
  buffer-uploaded flags, SRB state, frame counter) to
  `build/crash_dumps/render_state_*.log` on a structured check or via the
  existing crash handler. (b) The in-environment reproduction runs
  `OWzxSlicer.exe` with `OWZX_RHI_RENDERER=d3d12` for >=30s (multiple
  attempts), capturing the full `startup_diagnostics.log`. (c) Evidence is
  recorded: either a reproduced crash stack (root cause located) or an honest
  "does not reproduce in this headless environment" log.

## MIT: Seam Mitigations

- [ ] **MIT-01** (Phase 208, Seam A): `RhiViewportRenderer.cpp:526`'s
  out-of-pass `cb->resourceUpdate(readbackUpdates)` is replaced by stashing
  the readback batch in a `m_pendingReadbackUpdates` member, folded into the
  next on-screen `beginPass` (`:402`) 4th argument. After the change, the
  renderer contains NO direct `cb->resourceUpdate()` call outside a
  `beginPass` 4th-arg fold.

- [ ] **MIT-02** (Phase 209, Seam B): `uploadCameraUniform`
  (`:1559-1607`) replaces its three sub-range `updateDynamicBuffer` calls
  (`:1585`/`:1600`/`:1602`) with a single 80-byte write of a
  `CameraBlockPacked { QMatrix4x4 mvp; QVector3D gizmoCenter; float
  gizmoScale; }` struct, matching the std140 layout already documented at
  `:1592-1596`.

- [ ] **MIT-03** (Phase 210, Seam C): (a) The first N frames (N=3) for all
  canvas types unconditionally call
  `uploadCameraUniform(updates, DirtyCamera|DirtyGpu)`. (b) `initialize()`
  (`:21-33`) additionally resets `m_cameraUniformBufferUploaded`,
  `m_modelVertexBufferUploaded`, `m_highlightVertexBufferUploaded` (currently
  only `m_sceneBuffersUploaded` is reset, risking stale GPU memory after a
  swapchain rebuild). (c) The `m_thumbnailRenderPassDescriptor` raw-pointer
  lifetime (`:630` deleteLater) is audited and corrected if needed.
