# Phase 99 Plan: Wipe-Tower Geometry Gap Audit

## Objective

Create the canonical v4.4 wipe-tower geometry readback + rendering region map
(`99-GAP-MATRIX.md`) before any implementation. The matrix must let Phase 100-102
execute without re-discovering the current placeholder geometry path, the
upstream `Print::wipe_tower_data()` / `WipeTowerData` anchors, the post-slice
readback integration point, or the rendering-upgrade decision.

Specifically the matrix must freeze:

- The **placeholder geometry path**: `GizmoGeometry::buildWipeTowerVertices`
  (`src/core/rendering/GizmoGeometry.h:74` / `.cpp:449-499`, a 36-vertex
  rectangular prism with hardcoded color `{0.35, 0.60, 0.85, 0.50}`); the
  `RhiViewport` Q_PROPERTY defaults (`RhiViewport.h:54-59,181-192,304-309`,
  width=10/depth=10/height=50/x=100/z=25); the renderer buffer + upload/render
  path (`RhiViewportRenderer.h:57,75,129,143,152,177,216-222`,
  `RhiViewportRenderer.cpp:1064-1095 uploadWipeTowerBuffer`,
  `:1894-1908 renderWipeTower`); the parallel SoftwareViewport props
  (`SoftwareViewport.h:35-40,126-127,231-236`, `SoftwareViewport.cpp:207-251`).
- The **upstream source-truth anchors**: `Print.hpp:740-786` (`struct
  WipeTowerData`: `tool_changes`, `bbx` includes brim, `rib_offset`, optional
  `wipe_tower_mesh_data`, `depth`, `height`, `brim_width`, `position`, `width`),
  `Print.hpp:988-989` (`has_wipe_tower()`, `wipe_tower_data()`),
  `Print.hpp:1078-1080` (`get_wipe_tower_depth()`, `get_wipe_tower_bbx()`,
  `get_rib_offset()`), `3DScene.cpp:840-882 load_wipe_tower_preview`
  (`make_cube(width, depth, height)`), `3DScene.cpp:887-923
  load_real_wipe_tower_preview` (real mesh via `convex_hull_3d`).
- The **post-slice readback integration point** (WTAUDIT-02): the `Print`
  object lives in the `SliceService` worker lambda
  (`SliceService.cpp:508 receiver->activePrint_.store(&print, ...)` and
  `:584 print.process()`); slice results are delivered to the GUI thread via
  the worker lambda that emits `sliceFinished` at `SliceService.cpp:763`.
  `EditorViewModel` holds `sliceService_` (`EditorViewModel.h:860`) and is the
  bridge that pushes geometry into `RhiViewport`. The Prepare page's
  `GLViewport` (`PreparePage.qml:1648`) currently does NOT bind any wipe-tower
  Q_PROPERTY — the renderer reads its hardcoded defaults.
- The **rendering-upgrade decision** (WTAUDIT-02): Option A (dimensioned box
  from real `bbx`/`depth`/`height`/`position`/`width`) vs Option B (real mesh
  from optional `wipe_tower_mesh_data` via `convex_hull_3d`). The matrix must
  lock Option A as the v4.4 baseline and document Option B as a future upgrade.

This phase is read-only: it modifies documentation only and produces no
QML/C++ changes. Runtime wipe-tower visibility parity is owned by Phase 102;
Phase 99 does not claim it.

## Requirements

- `WTAUDIT-01`: The wipe-tower capture + rendering surface has a current
  inventory mapping the placeholder geometry path
  (`GizmoGeometry::buildWipeTowerVertices`, `RhiViewport` defaults
  10/10/50/100/25), the upstream `Print::wipe_tower_data()` / `WipeTowerData`
  struct anchors, the readback design (post-slice `Print` -> `RhiViewport`
  dims), and the rendering-upgrade decision (box dims vs real mesh) before
  implementation.
- `WTAUDIT-02`: The post-slice readback integration point (where
  `Print::wipe_tower_data()` becomes available after slicing, and how it flows
  to `RhiViewport` / `EditorViewModel`) and the rendering-upgrade approach
  (real mesh via `wipe_tower_mesh_data` / `convex_hull` vs dimensioned box
  from `bbx`/`depth`/`height`) are frozen as locked decisions before
  implementation.
