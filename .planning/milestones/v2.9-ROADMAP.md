## Milestone Summary

### Key Decisions

| Decision | Rationale | Outcome |
|---|---|---|
| Treat v2.9 as a stabilization/truth-reset milestone | Planning drift (v2.6 marked current while v2.7/v2.8 had landed) made the next large module unsafe to start. | Good — planning now agrees with git and code |
| Classify services Real/Hybrid/Mock/Blocked/Placeholder | `*Mock` class names no longer reflect implementation reality; needed an evidence-based vocabulary. | Good — vocabulary adopted across requirements/audits |
| Default to SoftwareViewport, gate OpenGL behind `OWZX_OPENGL` | Software viewport startup is the safer default; OpenGL path preserved for opt-in. | Good — INT-05 enforced by QmlUiAuditTests |
| Route calibration by stable ids, not list indexes | List-index routing is fragile across menu reordering. | Good — CAL-01/UI-03 stable |
| Defer full PartPlate/AssembleView to v3.0 | Large source-truth module; v2.9 first makes the baseline trustworthy. | Pending (v3.0 scope) |
| Defer upstream-compatible preset bundle to v3.1 | Requires a dedicated preset-system milestone. | Pending (v3.1 scope) |

### Issues Resolved

- Planning drift: `.planning` now agrees with git history through v2.8 and marks v2.9 complete.
- Source hygiene: targeted literal-escape (`\r\n`) artifacts and mojibake removed from touched active files; `SliceService.cpp.backup` removed.
- Calibration honesty: implemented PA/Flow/Temp modes are visible, deterministic, and test-asserted; unimplemented modes are explicit Pending/Blocked, not silent gaps.
- Integration confidence: SSDP/MQTT/FTP/camera paths now have deterministic protocol-level test coverage instead of relying on transport fallback confidence.
- UI honesty: disabled/no-op placeholder controls are now hidden or explicitly classified, not counted as feature completion.

- Upstream-compatible preset bundle and CreatePresetsDialog workflows.
- Repo-root build cruft and broad repo-wide mojibake scan (Phase 11 scoped to touched files).
