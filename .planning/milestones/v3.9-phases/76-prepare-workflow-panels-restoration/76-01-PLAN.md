# Plan 76-01: Workflow Panels Restoration

## Goal

Restore Prepare object list, plate strip, and slice status UI to the Phase 76
contract while preserving the existing C++ ViewModel gates and upstream-backed
interactions.

## Tasks

1. RED static audit
   - Add a focused `QmlUiAuditTests` method for Phase 76.
   - Assert that the floating viewport slice button is gone.
   - Assert compact object-list, plate-strip, and slice-readiness contracts.
   - Run the canonical verifier and capture the expected failing audit.

2. Object list restoration
   - Add explicit compact tree sizing properties.
   - Tighten object row, volume row, group header, indentation, and pills.
   - Keep selection, context menu, rename, drag/drop, printable, volume, and
     delete behavior wired to existing ViewModel calls.

3. Plate strip restoration
   - Reduce bottom strip height, card width, thumbnail size, and radius.
   - Add explicit text for missing/sliced/stale result states.
   - Preserve active plate selection, add plate, context menu, and drag/drop.

4. Slice status restoration
   - Remove the viewport floating `Slice` button from `GLToolbars.qml`.
   - Keep the `sliceRequested` signal available for compatibility even if no
     viewport control emits it.
   - Add a single backend-gated primary action property in `SliceProgress`.
   - Disable slice-all unless slice or result state makes it meaningful.

5. Verification and closeout
   - Run `QmlUiAuditTests.exe`.
   - Run `git diff --check`.
   - Run encoding guard.
   - Run the canonical verifier.
   - Write summary, verification, code review, and UI review artifacts.
   - Update ROADMAP/STATE completion markers if checks pass.

## Acceptance Criteria

- `OBJ-01`: object/volume list states are compact, stateful, and action-gated.
- `PLATEUI-01`: plate strip shows active/add/result states in compact form.
- `STATUS-01`: slice/cancel/export/readiness surfaces are honest and no
  floating viewport slice button remains.
