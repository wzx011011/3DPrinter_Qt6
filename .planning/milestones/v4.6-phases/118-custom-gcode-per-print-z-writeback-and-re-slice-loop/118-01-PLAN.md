# Phase 118 Plan: custom_gcode_per_print_z Writeback And Re-Slice Loop

## Objective
Close TICK-02 + TICK-03: wire PreviewViewModel tick CRUD into libslic3r's per-plate custom-gcode store (`model->plates_custom_gcodes`) and trigger a re-slice via `sliceService_->startSlice()` so the resulting G-code contains the user's markers. Phase 117 surfaced the UI; Phase 118 closes the loop.

## Critical: set_custom_gcode_per_print_z does NOT exist
Use direct field assignment (the BBS path): `model->curr_plate_index = idx; model->plates_custom_gcodes[idx] = info;` (`Model.hpp:1559-1570`). `cloneCurrentPlateModel()` copies the map (Model.cpp:82-83), so re-slice via `startSlice` sees the written codes.
