# Phase 117 Plan: IMSlider Integration And Tick Rendering

## Objective
Close TICK-01: make tick marks (pause / color-change / filament-change / custom-gcode / template) render on the Preview layer rail so a sliced G-code's layer events are visible, and make the right-click Add / Edit / Delete menus reachable in the running Preview. UI-integration only — write-back to `custom_gcode_per_print_z` and re-slice are Phase 118; new tick types + drag are Phase 119.

## Key Decision (from code research)
**Consolidate into the existing vertical `PreviewLayerRail.qml`** — do NOT relocate the horizontal orphan `LayerSlider.qml`. Upstream OrcaSlicer `IMSlider` is vertical (right-side rail); `PreviewLayerRail` is already vertical and working. Porting LayerSlider's tick Repeater + right-click menus into PreviewLayerRail keeps vertical source-truth alignment, avoids a duplicated slider, and leaves no orphaned component. The horizontal `LayerSlider.qml` is deleted (No-Deprecated-UI rule).

**CustomGcode color gap:** `LayerSlider.qml:101-108` has no `case 1` branch — CustomGcode ticks fall to `default` (gray). Upstream renders custom-gcode ticks distinctly. Add a `case 1:` branch using an existing distinct Theme token (not a new teal token — Theme has none; reuse `Theme.statusInfo` alternate or a documented distinct existing color). The executor picks the closest existing token and documents the choice.
