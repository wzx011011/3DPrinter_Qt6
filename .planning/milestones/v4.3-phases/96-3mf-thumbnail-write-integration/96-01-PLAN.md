# Phase 96 Plan: 3MF Thumbnail Write Integration

## Objective

Close the 3MF thumbnail write side. Phase 95 now delivers real captured pixels
(as a `QImage` via the preserved `lastThumbnailData`/`thumbnailCaptured` contract,
cached per plate on `PartPlate::m_thumbnail`). Phase 96 converts those pixels
into `Slic3r::ThumbnailData` (width/height/RGBA byte vector) and populates the
two write-side hooks that were explicitly deferred in v3.2 (Phase 30):

1. **THUMBWRITE-01 (per-plate, `PlateData::plate_thumbnail`):** `buildPlateDataList`
   (`ProjectServiceMock.cpp:5051`, the deferred comment at `:5089-5093`) populates
   each plate's `PlateData::plate_thumbnail` (`bbs_3mf.hpp:80`, a value
   `ThumbnailData` member) from the plate's cached `PartPlate::thumbnail()` so the
   writer emits the XML `<metadata thumbnail_file="Metadata/plate_N.png">`
   reference (`bbs_3mf.cpp:7987-7990`) per plate.

2. **THUMBWRITE-02 (project-level, `StoreParams::thumbnail_data`):** `saveProject`
   (`ProjectServiceMock.cpp:5100`, the deferred comment at `:5127-5135`) populates
   `StoreParams::thumbnail_data` (`bbs_3mf.hpp:235`, `vector<ThumbnailData*>`) so
   the writer iterates it (`bbs_3mf.cpp:6133-6143`) and writes the PNG bytes
   `Metadata/plate_N.png` via `_add_thumbnail_file_to_archive`
   (`bbs_3mf.cpp:6543-6607`).

3. **THUMBWRITE-03 (PNG path runs to completion):** with real RGBA pixels in
   `ThumbnailData::pixels`, `_add_thumbnail_file_to_archive` calls
   `tdefl_write_image_to_png_file_in_memory_ex(pixels.data(), w, h, 4, ...)`
   (`bbs_3mf.cpp:6548`) which returns non-null on real pixels →
   `mz_zip_writer_add_mem` (`bbs_3mf.cpp:6551`) succeeds → no exception, no silent
   skip. The v3.2 "throws a non-std exception" was the empty/mock-pixel path;
   populating real pixels makes THUMBWRITE-03 pass naturally.

The format symmetry is load-bearing: the READ side wraps `ThumbnailData::pixels`
as `QImage::Format_RGBA8888` (`ProjectServiceMock.cpp:5455-5466`), so the write
side MUST produce RGBA8888 bytes via `convertToFormat(Format_RGBA8888)` before
copying — otherwise the round-trip produces a corrupt/swizzled image.

Phase 96 owns write-side population only. Save-reload round-trip verification,
mock-generator removal, and final runtime visual proof are owned by Phases 97 and
98 respectively. Phase 96 routes the runtime save round-trip proof to Phase 97.

## Requirements

- `THUMBWRITE-01`: On project save, `ProjectServiceMock::saveProject` populates
  `PlateData::plate_thumbnail` (in `buildPlateDataList`) with real captured
  pixels, so per-plate thumbnails are written into the 3MF archive (closes the
  `ProjectServiceMock.cpp:5089-5093` deferred write).
- `THUMBWRITE-02`: On project save, `StoreParams::thumbnail_data` is populated
  with a real captured thumbnail so the project-level thumbnail is written into
  the 3MF archive (closes the `ProjectServiceMock.cpp:5127-5135` deferred write).
- `THUMBWRITE-03`: The upstream `store_bbs_3mf` PNG encoding path
  (`_add_thumbnail_file_to_archive`) runs to completion on the Qt6 pipeline
  without throwing the non-std exception previously seen on the mock pipeline.

      ```cpp
      #ifdef HAS_LIBSLIC3R
      // Phase 96 (THUMBWRITE-01/02): convert a captured QImage into a
      // Slic3r::ThumbnailData (width/height/RGBA bytes) for the 3MF writer.
      // Format symmetry: the READ side (ProjectServiceMock.cpp:5455) wraps
      // ThumbnailData::pixels as QImage::Format_RGBA8888, so the write side MUST
      // produce RGBA8888 bytes (convertToFormat) for a clean round-trip.
      // File-local (not a member) to keep bbs_3mf.hpp out of the public header,
      // matching the buildPlateDataList design (ProjectServiceMock.cpp:5048-5049).
      static Slic3r::ThumbnailData qimageToThumbnailData(const QImage &img)
      {
        Slic3r::ThumbnailData td;  // width=0/height=0/pixels empty (is_valid()==false)
        if (img.isNull())
          return td;  // empty cache → writer silently skips (bbs_3mf.cpp:6135,7987 is_valid guard)
        const QImage rgba = img.convertToFormat(QImage::Format_RGBA8888);
        const int w = rgba.width();
        const int h = rgba.height();
        if (w <= 0 || h <= 0)
          return td;
        td.width = static_cast<unsigned int>(w);
        td.height = static_cast<unsigned int>(h);
        td.pixels.resize(static_cast<size_t>(w) * static_cast<size_t>(h) * 4);
        // RGBA8888 has a tightly-packed stride of w*4 (no padding for this
        // format); copy scanline-by-scanline to be robust against any future
        // format that pads the stride.
        for (int y = 0; y < h; ++y) {
          const uchar *src = rgba.constScanLine(y);
          uchar *dst = td.pixels.data() + static_cast<size_t>(y) * w * 4;
          std::memcpy(dst, src, static_cast<size_t>(w) * 4);
        }
        return td;  // is_valid()==true (pixels.size()==4*w*h)
      }
      #endif // HAS_LIBSLIC3R
      ```
