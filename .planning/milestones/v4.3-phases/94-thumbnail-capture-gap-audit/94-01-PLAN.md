# Phase 94 Plan: Thumbnail Capture Gap Audit

## Objective

Create the canonical v4.3 thumbnail capture + 3MF writer gap matrix
(`94-GAP-MATRIX.md`) before any implementation. The matrix must let Phase 95-98
execute without re-discovering the current mock paths, the deferred write-side
gaps, the OrcaSlicer writer source anchors (`store_bbs_3mf` thumbnail path,
`_add_thumbnail_file_to_archive`, `PartPlate::store_to_3mf_structure`,
`_extract_from_archive`), the current Qt stub gap
(`RhiViewport.cpp:476-488 requestThumbnailCapture`,
`ProjectServiceMock.cpp:4242 generatePlateThumbnail`), the QRhi readback
decision, the MSAA resolve strategy, the render-thread capture queue design,
the destination thumbnail cache, the cleanup candidates, or the verification
routes.

This phase is read-only: it modifies documentation only and produces no
QML/C++ changes. Runtime pixel parity and visual proof are owned by Phase 98;
Phase 94 does not claim them.

## Requirements

- `THUMBAUDIT-01`: The thumbnail capture + 3MF writer surface has a current
  inventory mapping the mock paths (`generatePlateThumbnail` QPainter stub,
  `requestThumbnailCapture` solid-color stub), the deferred write-side gaps,
  upstream source anchors (`store_bbs_3mf` thumbnail path,
  `PartPlate::store_to_3mf_structure`), the QRhi readback approach, MSAA
  resolve strategy, and verification expectations.
- `THUMBAUDIT-02`: The QRhi readback design (offscreen `QRhiTexture`
  render-target + `readBackTexture`, vs live RT color attachment readback), the
  MSAA resolve approach (sample count > 1), and the render-thread capture
  request queue + QImage callback design are frozen as locked decisions before
  implementation.
