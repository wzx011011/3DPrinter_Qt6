### CL — Qt Crash / Behavior Fixups

- [ ] **CRASH-01** (Phase 180): Port upstream bb3 fixes to the Qt6 renderer/selection destructor chain. Two confirmed fixes:
  - **A1** (`5a53d2eb88`, upstream `GLCanvas3D.cpp` + `Selection.cpp`): GLCanvas destruction use-after-free — Qt6 equivalent is the RhiViewport destructor chain. The `is_closing()` guard pattern must be ported so that Selection/Scene teardown does not call back into a half-destroyed Plater/editor viewmodel.
  - **A2** (`d24e7f75ef`, upstream `Selection.cpp`): selecting the wipe tower / prime tower and then rotating or scaling crashed because the wipe-tower synthetic id (≥1000) was used as a ModelObject index. Qt6 equivalent is `EditorViewModel.cpp` + `src/core/rendering/ObjectPicking.{cpp,h}` — the transform loop must skip synthetic wipe-tower ids.

- [ ] **CRASH-02** (Phase 181): Port three upstream fixes to data + UI components:
  - **A4** (`4b7182b048`, upstream `PartPlate.cpp` + `.hpp`): deleting an instance left a stale instance id in `obj_to_instance_set`, causing an out-of-bounds scan. The fix makes `valid_instance()` const and adds bidirectional bounds checks at all 6 scan sites. Qt6 equivalent is `src/core/model/PartPlate.{cpp,h}`.
  - **A8** (`8e868118d2`, upstream `PresetBundleDialog.cpp`): the type field for process presets was rendered as `"presets"` instead of `"processes"`. Qt6 equivalent is `src/qml_gui/dialogs/ExportPresetBundleDialog.qml`.
  - **A9** (`b2adfb5c13`, upstream `NotificationManager.cpp`): shared-profile notification minimized-state + text wrapping. Qt6 equivalent is `src/qml_gui/components/NotificationCenter.qml`.

- [ ] **CRASH-03** (Phase 182): Port one editor-workflow fix and produce an architecture-divergence research report:
  - **A7** (`5ba5c6672d`, upstream `Plater.cpp` + `Preferences.cpp`): reopening a project failed STEP `reload_from_disk` because only the filename was stored. The fix adds a case-insensitive filename fallback match and a Preferences checkbox "preserve full source path". Qt6 equivalent is `src/core/viewmodels/EditorViewModel.cpp` (reload_from_disk) + `src/qml_gui/pages/PreferencesPage.qml`.
  - **RESEARCH.md deliverable** (no code changes required for the 4 items below — investigation only):
    - **A3** (`adc8763099`, upstream `GLGizmoMeasure.cpp`): SPHERE_2 gripper used `.first.feature` where it should have used `.second.feature`. Does Qt6's self-built `MeasureEngine.{cpp,h}` + `AssemblyMeasureGeometry.{cpp,h}` have an equivalent first/second feature confusion? Expected outcome: not applicable (Qt6 Measure is a rewrite).
    - **A5** (`59e655dd1d`, upstream `GUI_ObjectTable.cpp`): wxGrid dynamic_cast crash. Does Qt6 `ObjectList.qml` have an equivalent path? Expected outcome: not applicable (Qt6 has no wxGrid).
    - **A6** (`49fe64cb07`, upstream `Plater.cpp`): toolchanger/IDEX sidebar filament-region collapse boundary. Does Qt6 `FilamentSlot.qml` have equivalent geometry? Expected outcome: not applicable (different layout system).
    - **A10** (`33909a51cd`, upstream `3DScene.cpp` + 10 GLSL shaders): OpenGL stencil-buffer outline thickness. Does Qt6 RHI rendering have an equivalent bug? Expected outcome: not applicable (Qt6 uses RHI + self-built shaders, no gouraud/phong/ssao).
    - Each item in RESEARCH.md classified as "not applicable" / "deferred to follow-up" / "fixed in this phase" (last option only if the investigation surfaces an actual Qt6 bug).

### FEAT — Per-Extruder Config UI Bridge

- [ ] **FEAT-04** (Phase 183): Bridge bb3's nullable config types to the Qt6 UI. bb3 enabled `ConfigOptionFloatsNullable` / `ConfigOptionBoolsNullable` / `ConfigOptionFloatsOrPercentsNullable` for per-extruder speed/acceleration/jerk settings. The Qt6 ConfigOptionModel currently renders single values. Implement:
  - Recognition of nullable fields in `ConfigOptionModel` (the `opt->nullable` flag is already read by Qt6 — make it drive UI mode).
  - A per-extruder editor surface (one row or column per extruder) for nullable fields.
  - Mapping to upstream `ConfigManipulation.cpp` multi-extruder UI logic for behavioral parity.
  - **Open question**: estimated 800-1200 lines. May split into 183a (data layer: ConfigOptionModel + ConfigViewModel nullable-aware getters/setters) + 183b (UI layer: per-extruder editor QML) if scope is large.

### I18N — Translation Long Tail Round 2
