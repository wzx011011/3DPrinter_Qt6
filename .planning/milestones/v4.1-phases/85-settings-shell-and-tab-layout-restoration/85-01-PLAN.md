# Plan 85-01: Restore Settings Dialog Shell And Tabs

## Goal

Restore the screenshot-visible printer/material/process settings dialog shell:
native-title non-modal window, compact preset/action row, compact tabs, clean
UTF-8 labels, no off-design left group sidebar, and preserved settings entry
behavior.

## Must Haves

- D-01: preserve independent 736x593 `ApplicationWindow`.
- D-02: remove duplicate in-row close and keep dirty-close guard.
- D-03: restore clean titles: `打印机设置`, `材料设置`, `工艺设置`.
- D-04: keep one main content pane with right scrollbar and no visible left
  group navigation.
- D-05: replace wide search/text-button top row with preset combo plus
  compact actions.
- D-06: preserve preset selection, save/save-as, search, advanced mode, dirty
  status, and compatibility status routing.
- D-07: search becomes a compact reveal affordance.
- D-08: advanced/simple mode becomes compact.
- D-09: reuse existing icons/controls.
- D-10: printer tab labels/order match screenshot.
- D-11: material tab labels/order match screenshot.
- D-12: process uses source-parity tabs and same shell.
- D-13: keep stable internal keys where possible.
- D-14: preserve three `SettingsDialog` instances in `main.qml`.
- D-15: preserve Prepare/sidebar settings buttons; clean only touched
  settings-related text.
- D-16: do not route topbar preferences into parameter settings.
- D-17: match dark/compact/cyan settings style locally.
- D-18: keep dense operational layout, no cards inside cards.
- D-19: all touched visible strings are clean UTF-8 and `qsTr()` wrapped.
- D-20: update source audits for no visible group sidebar, clean labels,
  compact actions, no duplicate close, and preserved dispatch.
- D-21: run static/source checks and targeted tests where available; final
  screenshots stay Phase 88.

## Tasks

### 85-01-01: Update Source Audits First

Modify `tests/QmlUiAuditTests.cpp`:

- replace the old `settingsDialogFiltersByTabAndGroup()` invariant with a
  Phase 85 shell invariant;
- assert `SettingsDialog.qml` contains clean titles and exact printer/material
  tab labels;
- assert mojibake title/tab fragments are absent from the restored shell;
- assert `GroupNavSidebar {`, `selectedGroup`, and `filterIndicesByGroup` are
  absent from `SettingsDialog.qml`;
- assert top-row text `Save` / `Save As...` `CxButton` usage is absent;
- assert compact icon buttons, `requestSaveAndMaybeClose`, `saveAsDialog.open`,
  search reveal state, and `advancedMode` remain wired;
- keep `settingsDialogMainQmlDispatchStructural()` for `main.qml` dispatch.

Leave `ViewModelSmokeTests.cpp` unchanged unless implementation changes
backend entry behavior.

### 85-01-02: Recompose `SettingsDialog.qml` Shell

Patch `src/qml_gui/dialogs/SettingsDialog.qml`:

- restore clean `title` text;
- restore printer/material/process tab labels while preserving keys;
- remove `selectedGroup` shell state and group filtering;
- keep page filtering through `filterIndicesByPage`;
- replace the permanent wide search field with a compact search icon/reveal
  field;
- replace text `Save` and `Save As...` buttons with compact `CxIconButton`
  actions and tooltips;
- keep dirty and compatibility indicators compact;
- remove the manual in-row close button and rely on native close plus Escape;
- keep `SavePresetDialog`, `UnsavedChangesDialog`, `requestSaveAndMaybeClose`,
  and `onClosing` dirty guard intact.

### 85-01-03: Preserve Entrypoints

Check `src/qml_gui/main.qml` and `src/qml_gui/panels/LeftSidebar.qml`:

- preserve `printerSettingsDialog`, `materialSettingsDialog`, and
  `processSettingsDialog`;
- preserve `backend.forwardSettingsRequest("printer")`,
  `backend.forwardSettingsRequest("filament")`, and
  `backend.forwardSettingsRequest("process")`;
- avoid broad topbar preference routing changes.

Only edit these files if Phase 85 audits show touched visible settings text or
dispatch has drifted.
