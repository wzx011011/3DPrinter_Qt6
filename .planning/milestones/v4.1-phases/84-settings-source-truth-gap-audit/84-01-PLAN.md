# Phase 84 Plan: Settings Source-Truth Gap Audit

## Objective

Create the canonical v4.1 settings gap matrix for printer, material, and
process settings dialogs. The matrix must let Phase 85-88 restore the dialogs
without re-discovering visual truth, upstream behavior truth, current Qt
targets, Phase 56 residuals, owner phases, or verification routes.

## Requirements

- `SETAUDIT-01`: Map user-facing printer, material, and process settings work
  to screenshot-visible regions, OrcaSlicer source files, Qt targets,
  replacement decisions, and verification evidence.
- `SETAUDIT-02`: Reconcile Phase 56 residual settings visual-UAT items into
      v4.1 requirements or explicitly close/defer them with evidence.

<must_haves>

  <truth id="D-09">Phase 56 deferred visual parity, typed-control visual rendering, and non-modal cross-window live edit evidence are reopened into v4.1 ownership.</truth>

  <truth id="D-15">Mojibake cleanup is mandatory for settings-visible strings.</truth>
  <truth id="D-16">Phase 84 verification is source/document focused: gap-matrix completeness, requirement coverage, `git diff --check`, and encoding guard.</truth>
  <truth id="D-17">Phase 88 must record printer/material/process visual evidence, with printer/material compared against target screenshots.</truth>
  <truth id="D-18">Future QML audits must cover clean text, dead controls, entry-point operability, option-control binding, and stale path cleanup.</truth>
