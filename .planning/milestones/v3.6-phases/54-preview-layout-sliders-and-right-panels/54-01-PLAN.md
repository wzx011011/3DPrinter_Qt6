# Phase 54 Plan - Preview Layout, Sliders, and Right Panels

## Objective

Complete the Preview page shell and state surfaces around the existing QRhi/D3D11 G-code renderer without regressing draw stability.

## Task 1 - Extend PreviewViewModel Data Surface

Files:

- `src/core/viewmodels/PreviewViewModel.h`
- `src/core/viewmodels/PreviewViewModel.cpp`

Steps:

1. Add read-only properties for preview readiness/status and the active G-code text window:
   - `previewReady`
   - `previewStatusText`
   - `currentLayerLabel`
   - `currentMoveLabel`
   - `plateSummary`
   - `warningSummary`
   - `gcodeLineCount`
   - `currentGcodeLine`
   - `gcodeLines`
2. Store a bounded `QVariantList` of G-code line rows while parsing the active file.
3. Keep the text window bounded around the current move to avoid loading a huge full-file model into QML.
4. Update the current G-code line when `setCurrentMove()` or playback changes.
5. Reset all text/status fields in `resetPreviewState()`.

Verification:

- `ViewModelSmokeTests` or `E2EWorkflowTests` verifies `gcodeLines`, `currentGcodeLine`, and status labels after loading a temporary G-code fixture.
- Existing tests still verify `gcodePreviewData()` stays non-empty after slider/toggle interactions.
