# Plan 52-01: C++ Foundation -- Slice Invalidation Wiring + Staleness Q_PROPERTYs + Settings Signal Forward

<objective>
Land the C++ foundation that the Phase 52 sidebar depends on, before any QML is
touched (Plan 52-02) or tested (Plan 52-03). Three things, all C++:

1. **PREPSB-05 (CRITICAL gap fix):** Expose slice staleness to QML on
   `EditorViewModel` via two new Q_PROPERTYs (`stalePlateIndices`,
   `hasStaleSliceResults`), backed by the existing `m_stalePlateIndices` set
   (EditorViewModel.h:759). Then, in `BackendContext` (the composition root that
   owns both viewmodels), wire `ConfigViewModel::stateChanged` ->
   `EditorViewModel::invalidateAllSliceResults()` so that changing a preset /
   scope / option actually invalidates previously-sliced/previewed/exported
   results. Today the invalidator exists but has NO preset-change caller, so a
   user could change a filament preset and export G-code based on the OLD
   preset -- a silent correctness bug.

2. **PREPSB-02 (interim settings entry point):** Add a
   `settingsRequested(QString category)` signal to `BackendContext` and a
   `forwardSettingsRequest(QString category)` slot whose body is an honest
   qInfo log ("settings dialog pending Phase 56"). The QML "Setting" button
   (Plan 52-02) emits this; Phase 56 connects the real independent dialog.

3. Keep `canSlice` (Phase 51 shell gate) SEPARATE from staleness -- canSlice
   reflects "is there something to slice"; staleness reflects "is the existing
   slice result out of date". They are NOT conflated.

This plan does NOT touch any `.qml` file (Plan 52-02), tests (Plan 52-03),
`PresetListModel`, settings dialog internals (Phase 56), or legacy page removal
(Phase 57).
</objective>

```cpp
QVariantList EditorViewModel::stalePlateIndices() const
{
  // Phase 52 PREPSB-05: surface the stale-plate set to QML so Preview/Export
  // can show a "stale -- reslice" indicator and disable export of stale results.
  // m_stalePlateIndices is maintained by invalidateSliceResultsForPlate /
  // invalidateAllSliceResults and cleared on a successful reslice.
  QVariantList list;
  for (int plateIndex : m_stalePlateIndices)
    list.append(plateIndex);
  return list;

bool EditorViewModel::hasStaleSliceResults() const

  // True when any previously-sliced plate result no longer matches current
  // inputs (a preset/scope/option changed after slicing). Kept SEPARATE from

  // existing result out of date".
  return !m_stalePlateIndices.isEmpty();
}
```
