# Plan 51-02: QML Shell Wiring — Undo/Redo Gate Fix + Action Bindings

<objective>
Wire the shell-level action buttons in `BBLTopbar.qml` to the new C++ gate properties added in Plan 51-01, fixing the concrete UX bug that Undo/Redo are clickable when the undo stack is empty (today they are page-gated only). After this plan, the Undo/Redo buttons (both the toolbar icons AND the Edit-menu items) bind `enabled` to BOTH the page gate (`currentPage === tp3DEditor`) AND the shell-level `canUndo`/`canRedo`, so SHELL-03 ("driven by C++ state, not QML-only conditions") is satisfied at the binding level. The Save button gains the `canSave` gate (disabled while slicing). The Slice action gains the `canSlice` gate. No new controls, no new actions, no new product behavior — only rebinding existing screenshot-visible controls to C++ state (source-truth rule). This satisfies the BBLTopbar portion of SHELL-01 (top shell with high-priority workflow actions) and the QML-side half of SHELL-03.

This plan depends on Plan 51-01 (the C++ gate properties must exist before QML can bind to them). It does NOT touch tests, Toolbar.qml removal, or the inventory (Plan 51-03). It does NOT touch navigation/state-preservation (already upstream-aligned and verified in Plan 51-03/51-01) — main.qml's StackLayout navigation is explicitly out of scope to MODIFY beyond what already works.
</objective>
