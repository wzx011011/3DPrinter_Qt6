## Notable finding: v5.3 parity audit was wrong about Simplify

The v5.3 feature-parity audit flagged Simplify as "explicit stub — not yet
implemented". Direct code investigation (Phase 176) revealed the real simplify
pipeline already existed end-to-end (simplifySelected → simplifyObject →
its_quadric_edge_collapse). The stub was a DUPLICATE method simplifyMeshSelected
that PreparePage/ObjectList were calling instead of the real method. Phase 176
fixed the call-chain delegation rather than re-implementing the math.
