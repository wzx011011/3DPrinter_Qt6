# 子 Agent 委托策略

积极使用子 agent 执行独立任务，保持主上下文精简。**批量模式下子 agent 委托不是可选优化，而是必要要求。**

## 委托表

| 任务类型 | Agent 类型 | 说明 |
|----------|------------|------|
| 上游源码分析 | `Explore` | 分析 `third_party/CrealityPrint` 中的行为，返回关键发现 |
| 代码搜索 | `Explore` | 搜索 QML/C++ 文件定位目标模块 |
| 代码实现 | `general-purpose` | 独立实现迁移代码，完成后返回 diff 摘要 |
| 构建验证 | `general-purpose` | 运行 `auto_verify_with_vcvars.ps1`，返回通过/失败 |
| 文档更新 | `general-purpose` | 更新追踪文档对应条目，返回更新摘要 |

## 执行模式

```
主 Agent (精简上下文，仅做任务选择+汇总)
|-- Agent 1: 分析上游源码 -> 返回关键 API/行为摘要
|-- Agent 2: 实现迁移代码（含读取目标文件） -> 返回修改摘要
|-- Agent 3: 构建验证 -> 返回通过/失败
|-- Agent 4: 更新追踪文档 -> 返回更新摘要
```

## 规则

- **主 agent 只负责**：读取任务队列、选择任务、调度子 agent、汇总结果
- **凡是涉及文件读取、代码修改、构建执行的操作，都必须委托给子 agent**，不得在主上下文中展开
- 子 agent 返回**精简摘要**（不超过 10 行），不返回完整代码或长日志
- **批量模式**：主 agent 每个任务只消耗约 5-10 行子 agent 返回结果，而非数百行工具输出
- 如果子 agent 失败，主 agent 可启动修复子 agent，但不应自己展开文件内容
