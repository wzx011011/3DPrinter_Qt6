<objective>
Inject user-selected preset configuration into the slice engine so that slicing uses real user parameters (layer height, infill, speed, supports, temperatures) instead of factory defaults. Wire ConfigViewModel's merged 3-tier preset hierarchy through BackendContext into SliceService's DynamicPrintConfig before Print::apply().

Purpose: The single biggest blocker for E2E workflow is that SliceService creates config from `full_print_config()` factory defaults (SliceService.cpp:228). User-selected printer/filament/process presets in ConfigViewModel never reach the slice engine. The slice succeeds but ignores all user parameters.

Output: Preset-injected slicing, E2E test coverage.
</objective>
