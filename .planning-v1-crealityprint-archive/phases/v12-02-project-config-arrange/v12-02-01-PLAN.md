# PLAN: v12-02-01 — Propagate Project Config to UI, Use Real Bed Shape for Arrange

## Context

When loading a 3MF project, the embedded DynamicPrintConfig is parsed by libslic3r but discarded. Opening a saved project always reverts to default preset values. Additionally, `arrangeObjects()` uses `Slic3r::InfiniteBed` which ignores real bed boundaries, so arranged objects can end up outside the printable area.

## Tasks

**T1. Add `projectConfigLoaded` signal to ProjectServiceMock**
- File: `src/core/services/ProjectServiceMock.h`
- Add signal: `void projectConfigLoaded(const QHash<QString, QVariant> &config);`
- File: `src/core/services/ProjectServiceMock.cpp`
- In `loadProject()` main-thread success callback (~line 4453, before `emit projectChanged()`):
  - Convert the captured `loadedConfig` (DynamicPrintConfig) to QHash<QString, QVariant>
  - Emit `projectConfigLoaded(convertedConfig)`
- Conversion: iterate `loadedConfig.keys()`, use `config.opt_serialize()` or direct type dispatch (Float/Int/Bool/String/Enum) to build QVariant values
- Note: `loadedConfig` must be captured by value in the main-thread lambda (it's declared in the background thread lambda at line 4253, needs explicit copy)

**T2. Add `applyProjectConfig()` method to ConfigViewModel**
- File: `src/core/viewmodels/ConfigViewModel.h`
- Add: `Q_INVOKABLE void applyProjectConfig(const QHash<QString, QVariant> &config);`
- File: `src/core/viewmodels/ConfigViewModel.cpp`

  1. Iterate config keys, match known keys to setters (layer_height → setLayerHeight, etc.)
  2. For printer/filament/print preset names: call setCurrentPrinterPreset/setCurrentFilamentPreset/setCurrentPrintPreset

**T3. Wire signal to slot in BackendContext**

### Wave 2: Real Bed Shape for Arrange

**T4. Add `printableArea` parameter to `arrangeObjects()`**
- File: `src/core/services/ProjectServiceMock.h` — update signature:
  ```cpp
  Q_INVOKABLE bool arrangeObjects(float spacing, bool allowRotation, bool alignY,
                                    const QString &printableArea = QString());
  ```
- File: `src/core/services/ProjectServiceMock.cpp` ~line 2049
- When `printableArea` is non-empty, parse "x1,y1;x2,y2;..." polygon format
- Build `Slic3r::Points` bed_shape from parsed coordinates
- Use `Slic3r::BoundingBox bed_bb(bed_shape)` then create appropriate bed type for `arrange_objects()`
- When `printableArea` is empty, fall back to `InfiniteBed` (current behavior)
