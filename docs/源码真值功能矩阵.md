# OrcaSlicer 源码真值功能矩阵

> 更新日期：2026-03-10
> 上游真值基线：见 [docs/源码真值基线.md](源码真值基线.md)
> 用途：建立“上游模块 -> Qt6 模块 -> 当前状态 -> 验收方式”的统一矩阵，作为后续所有迁移开发的执行入口。

## 使用说明

- 本矩阵以 `third_party/OrcaSlicer` 冻结基线为准。
- “当前状态”只描述当前 Qt6 承接情况，不代表最终功能完成度。
- `Qt6 当前锚点` 允许写“当前真实存在的文件”或“后续应迁入的目标模块”。
- 若当前代码现实与旧文档不一致，以本矩阵和实际文件状态为准。

状态口径：

- `[x]` 已完成上游对照并验证
- `[-]` 已有基础承接或局部闭环，但未完成完整对照
- `[ ]` 未开始或仅有占位能力

## 当前结论

- `P0.2` 所要求的矩阵已经建立，可以作为后续开发的统一入口文档。
- 该矩阵已覆盖当前任务体系中的主要业务域，并给出上游模块、Qt6 当前锚点和验收方式。
- 后续如果增加更细粒度子任务，应优先扩展本矩阵，而不是重新发明新的任务口径。

## 已知现实差异

- 当前服务层中真实存在的是 `ProjectServiceMock.*`、`PresetServiceMock.*`、`DeviceServiceMock.*` 等文件；旧文档中部分“真实服务”命名与代码现实存在偏差。
- 因此矩阵中的 `Qt6 当前锚点` 采用“当前真实文件 + 未来目标模块”并行描述，不再假设不存在的文件已经落地。
