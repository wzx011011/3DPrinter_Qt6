# v3.6 UI Inventory — Screenshot & Source-Truth Contract

> Living document. Statuses updated by Phases 51–57. Region IDs are immutable.

This is the single long-lived canonical inventory for the v3.6 screenshot-driven
restoration milestone. It maps every screenshot-visible region across the four
supplied screenshots to a frozen 9-column schema, with per-cluster upstream
coverage assertions and a greppable aggregate cleanup checklist. It is consumed
by every downstream v3.6 phase (51–57) and verified by Phase 58 (VERIFY-01).

Region identity below is triangulated from (a) the Qt6/QML page structure
(`PreviewPage.qml`, `PreparePage.qml`, `LeftSidebar.qml`, `SettingsPage.qml`,
which implement the screenshot layouts 1:1), (b) the analyzed settings dialog
shells, and (c) upstream OrcaSlicer `Plater` / `GUI_Preview` / `Tab` structure.
The two large page screenshots (`准备页.png`, `预览页.png`) could not be
parsed by the local vision pipeline, so their region decomposition follows the
QML structural reference, which the research confirms is the high-confidence
1:1 region source.

## 0. How to read this document

### Status vocabulary (FROZEN — 7 terms, no others)

Every `status` cell uses exactly one of:

`Real` | `Hybrid` | `Mock` | `Blocked` | `Placeholder` | `Superseded` | `Missing`

- `Real` — implemented against upstream behavior with real data, verified.
- `Hybrid` — partially real, partially mock/simplified; live but incomplete.
- `Mock` — placeholder data or stubbed behavior behind a real-looking control.
- `Blocked` — depends on an unavailable dependency (e.g. OpenVDB, FFmpeg) and
  cannot be completed in v3.6.
- `Placeholder` — UI region exists but no real behavior wired yet.
- `Superseded` — superseded by another region/implementation (legacy).
- `Missing` — the Qt target does not exist yet; no current file to map.
  (The Owzx-added 7th term, required for restored regions that have no Qt file.)

### Verification-method vocabulary (FROZEN — 6 terms, no free text)

Every `verification` cell uses exactly one of:

`automated-test` | `deterministic-harness` | `manual-visual` | `manual-uat-checklist` | `build-only` | `upstream-parity-audit`

- `automated-test` — covered by a Qt Test / CTest automated test.
- `deterministic-harness` — checked by a deterministic doc/schema harness
  (e.g. an inventory audit modeled on `QmlUiAuditTests.cpp`).
- `manual-visual` — verified by visual comparison to the screenshot.
- `manual-uat-checklist` — verified by a manual UAT checklist run.
- `build-only` — verified only by successful build + load (no behavior test).
- `upstream-parity-audit` — verified by an upstream symbol/behavior audit.

### Column schema (FROZEN — 9 columns, fixed order, byte-identical header)

Every region table uses this exact header row:

```
| region_id | region_name | visible_controls | qt_target | upstream_source | status | verification | modify_or_replace | cleanup |
```

Column semantics:

1. `region_id` — ASCII-only, page-prefixed, stable identifier matching
   `^[A-Z]+-[A-Z0-9]+$` (ONE dash segment after the prefix; no slashes, no
   unicode, no multi-dash). Never reused across screenshots.
2. `region_name` — bilingual display name: English primary, Chinese in
   parentheses where useful (Chinese is permitted ONLY here and in the four
   screenshot filenames).
3. `visible_controls` — the concrete controls/elements visible in the region
   on the screenshot.
4. `qt_target` — the current Qt6/QML file/component that renders this region,
   or `Missing` when no target exists yet.
5. `upstream_source` — file-level + optional symbol reference into upstream
   OrcaSlicer (`Plater.cpp Sidebar::*`). NEVER line numbers (they drift vs the
   v7.0.1 lock). A blank upstream is NOT allowed; if a region has no upstream
   match the cell MUST read `none-mapped (Owzx-only)`.
6. `status` — one of the 7 FROZEN status terms above.
7. `verification` — one of the 6 FROZEN verification terms above.
8. `modify_or_replace` — one of `modify` / `replace` / `missing` (see §6
   decision rules). Mirrored in the §6 summary table.
9. `cleanup` — `n/a` for modify/missing decisions, or a machine-grep-friendly
   inline list of removal targets for replace decisions (see §7 aggregate).

### Region-ID convention

- ASCII-only, page-prefixed, stable: `PREP-*` (Prepare), `PREV-*` (Preview),
  `SETPRINT-*` (Printer settings), `SETMAT-*` (Material settings).
- Format `^[A-Z]+-[A-Z0-9]+$`: exactly ONE dash, a single uppercase-or-digit
  segment after the prefix. Multi-dash and slash forms are forbidden; use the
  single-dash concatenation instead (for example the floating gizmo panel is
  `PREP-GIZMOFLOAT` and the view-options control is `PREP-VIEWOPTS`).
- Region IDs are immutable once published; statuses mutate as Phases 51–57 land.

### Upstream reference convention

- File-level path under `third_party/OrcaSlicer/` + optional symbol/region,
  e.g. `Plater.cpp Sidebar::*`, `GCodeViewer.cpp Legend::*`.
- NO line numbers (drift vs the v7.0.1 lock).
- A blank upstream is forbidden; an Owzx-only region must read
  `none-mapped (Owzx-only)`.
