# 批量模式停止条件

当 `$ARGUMENTS` 包含 `all` 时，连续执行直到满足以下任一条件：

- `docs/源码对照迁移任务追踪.md` 中**全局**不存在可继续推进的未完成任务（所有 P0-P10 均无 `[-]` 或 `[ ]`）
- **全局扫描所有节后**，所有剩余任务都缺少清晰的上游映射或被外部依赖阻塞，且需要先补任务文档或等待用户决策
- 构建验证失败，且在当前调用中无法安全修复
- **全局**出现外部依赖、环境、权限或构建问题，导致**所有**剩余任务都无法继续可靠推进
- 用户显式要求停止

## 关键约束：跨节跳转

**停止条件必须全局评估，不能只看当前节。** 具体规则：

- 如果当前节（如 P2.7）的剩余任务都被外部依赖阻塞，**必须跳到下一节**（如 P2.8、P3 等），而不是停止
- "所有剩余任务都被阻塞"意味着 P0-P10 **每一节**的每个 `[-]` 和 `[ ]` 任务都无法推进
- 单个任务被阻塞（如需要 CGAL）**不是**停止理由，应跳过该任务选择下一个

## 连续执行约束

- 目标是"持续循环直到收敛或遇到停止条件"，不是无条件死循环
- 不要为了追求连续执行而跳过验证、跳过文档更新、或跨越不清晰的任务边界
- 每完成一个任务都要**全局重新评估**优先级，不能机械地沿用上一个任务的上下文
- 默认每完成 2 个任务运行一次 `scripts/auto_verify_with_vcvars.ps1`；如果单个任务已经影响运行时行为，则该任务完成后立即验证
- 当连续执行的工作量已经很大时，应在阶段性输出中明确报告已完成任务、当前验证状态和下一个候选任务
- **每完成 2 个任务输出一次恢复点摘要**（见 output-format.md §4），确保 compact 后可恢复
